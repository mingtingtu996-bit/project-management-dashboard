## 七、技术说明

### 定时任务实现方式

**使用技术**:
- Node.js `setTimeout` / `setInterval`
- Supabase JS Client
- TypeScript

**设计特点**:
1. **独立进程**: 调度器作为独立进程运行，不与 API 服务器耦合
2. **优雅退出**: 支持 SIGINT/SIGTERM 信号，正确清理资源
3. **执行日志**: 每次执行都记录到 `job_execution_logs` 表
4. **错误处理**: 单个任务失败不影响其他任务

### 数据库依赖

**风险统计任务依赖**:
- `projects` 表 - 获取活跃项目列表
- `risks` 表 - 统计风险数据
- `risk_statistics` 表 - 存储每日快照
- `job_execution_logs` 表 - 记录执行日志

**证照预警任务依赖**:
- `pre_milestones` 表 - 读取证照数据 ⚠️ **当前缺失**
- `warnings` 表 - 创建/更新预警记录

### 时间安排

**错峰执行**:
- 02:00 - 风险统计任务
- 02:30 - 自动预警服务（已禁用）
- 03:00 - 前期证照预警任务

**错峰原因**: 避免数据库并发压力，确保任务不会相互干扰
