## 严重问题汇总（需优先修复）

### 1. 🔴 缺少认证授权机制（安全/后端）

**影响**: 所有API端点公开可访问，任何人都可以增删改查项目数据

**位置**:
- `server/src/index.ts` (第31-50行)
- 所有路由文件

**修复建议**:
```typescript
// 添加JWT认证中间件
app.use('/api', authMiddleware)
app.use('/api/projects', projectsRouter)
```

---

### 2. 🔴 权限系统是空壳（前端）

**影响**: 权限控制完全失效，所有用户都具有管理员权限

**位置**:
- `client/src/components/PermissionGuard.tsx` (第31行) - 硬编码 `'admin'`
- `client/src/hooks/usePermissions.ts` (第35行) - 默认返回 `'editor'`

**修复建议**:
```typescript
// 连接真实的权限查询
const { permissionLevel } = usePermissions()
const currentRole = permissionLevel // 不是硬编码
```

---

### 3. 🔴 GET /:id 接口全表扫描性能问题（后端/性能）

**影响**: 数据量大时性能线性下降，浪费带宽和内存

**位置**:
- `server/src/routes/tasks.ts` (第83-84行)
- `server/src/routes/risks.ts` (第83-84行)
- `server/src/routes/milestones.ts` (第83-84行)

**当前代码**:
```typescript
const tasks = await supabase.getTasks()  // 加载所有任务
const task = tasks.find(t => t.id === id)  // 内存过滤
```

**修复建议**:
```typescript
const { data: task } = await supabase
  .from('tasks')
  .select('*')
  .eq('id', id)
  .single()
```

---

### 4. 🔴 Strict 模式未启用（TypeScript）

**影响**: 类型检查功能大幅减弱，类型安全性降低

**位置**:
- `client/tsconfig.json` (第14行)
- `server/tsconfig.json` (第7行)

**修复建议**:
```json
{
  "compilerOptions": {
    "strict": true,
    "noImplicitAny": true,
    "strictNullChecks": true
  }
}
```

---

### 5. 🔴 敏感信息泄露风险（安全）

**影响**: Supabase密钥直接暴露在前端环境变量

**位置**: `.env` 文件

**修复建议**:
- 使用后端代理访问Supabase
- 前端不直接访问数据库

---

### 6. 🔴 Dashboard页面每次渲染都重新计算统计数据（性能）

**影响**: 数据量大时（100+任务），filter操作消耗显著计算资源

**位置**: `client/src/pages/Dashboard.tsx` (第29-36行)

**修复建议**:
```typescript
const stats = useMemo(() => {
  const totalTasks = tasks.length
  const completedTasks = tasks.filter(t => t.status === 'completed').length
  // ... 其他计算
  return { totalTasks, completedTasks, /* ... */ }
}, [tasks, risks, milestones])
```

---

### 7. 🔴 版本冲突处理逻辑不完整（前端）

**影响**: 检测到版本冲突时只是简单继续更新，没有正确处理冲突

**位置**: `client/src/pages/GanttView.tsx` (第228-236行)

**修复建议**:
```typescript
if (updated) {
  updateTask(editingTask.id, { ...editingTask, ...taskData })
} else {
  // 显示冲突解决对话框
  setConflictData({ local: editingTask, incoming: taskData })
  setShowConflictDialog(true)
}
```

---

### 8. 🔴 `any` 类型滥用（TypeScript）

**影响**: 绕过类型检查，降低代码类型安全性

**位置**: 
- `client/src/hooks/useConflictDetection.ts` (多处)
- `client/src/lib/storageService.ts` (第611, 650行)
- 所有路由文件的 `catch (error: any)`

**修复建议**:
```typescript
// 使用 unknown 替代 any
catch (error: unknown) {
  if (error instanceof Error) {
    // 处理错误
  }
}
```

---

### 9. 🔴 useEffect依赖不完整（前端/性能）

**影响**: 可能导致闭包陷阱或无限循环

**位置**:
- `client/src/pages/ProjectList.tsx` (第31-33行)
- `client/src/pages/ProjectDetail.tsx` (第42-46行)

**修复建议**:
```typescript
useEffect(() => {
  if (id) {
    loadProjectData()
  }
}, [id, loadProjectData])  // 添加loadProjectData依赖
```

---

### 10. 🔴 CORS配置过于宽松（安全）

**影响**: 允许所有来源的跨域请求

**位置**: `server/src/index.ts` (第31行)

**修复建议**:
```typescript
app.use(cors({
  origin: ['https://yourdomain.com'],
  credentials: true
}))
```

---

### 11. 🔴 内存泄漏风险 - storageService事件监听未清理（前端/性能）

**影响**: 组件卸载时事件监听未清理，导致内存泄漏

**位置**: `client/src/lib/storageService.ts`

**修复建议**:
```typescript
useEffect(() => {
  const unsubscribe = storageService.on('change', handler)
  return () => unsubscribe()  // 清理监听
}, [])
```

---

### 12. 🔴 XSS防护不足（安全）

**影响**: 前端未使用内容安全策略(CSP)

**修复建议**:
```html
<meta http-equiv="Content-Security-Policy" 
      content="default-src 'self'; script-src 'self'">
```

---

### 13. 🔴 输入验证不完整（安全/后端）

**影响**: 部分端点缺少参数验证

**位置**: 多个路由文件

**修复建议**:
```typescript
import { z } from 'zod'

const createTaskSchema = z.object({
  title: z.string().min(1).max(200),
  status: z.enum(['pending', 'in_progress', 'completed'])
})
```

---

### 14. 🔴 缺少错误边界处理（前端）

**影响**: 组件崩溃会导致整个应用白屏

**修复建议**:
```typescript
// 添加ErrorBoundary组件
class ErrorBoundary extends React.Component {
  componentDidCatch(error, errorInfo) {
    // 记录错误
  }
  render() {
    if (this.state.hasError) {
      return <FallbackUI />
    }
    return this.props.children
  }
}
```
