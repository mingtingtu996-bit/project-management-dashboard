## 🔴 P0 — 阻断上线（必须修，否则无法运行）

### 1. 环境配置未填写
- **文件**：`server/.env`
- **问题**：`SUPABASE_URL` / `SUPABASE_SERVICE_KEY` 为占位符，服务启动即退出
- **影响**：100% 阻断，任何接口都跑不了

- **文件**：`client/.env` 中 `VITE_STORAGE_MODE=local`
- **问题**：数据写入 LocalStorage，多用户隔离，部署后数据丢失
- **修复**：改为 `VITE_STORAGE_MODE=supabase`

### 3. 生产环境变量未配置
- **文件**：`server/.env`
- **问题**：`NODE_ENV=development`、`CORS_ORIGIN=http://localhost:5173`
- **修复**：改为 `NODE_ENV=production`，`CORS_ORIGIN` 改为实际部署域名

### 5. 乐观锁非原子性问题 ✅ 已修复
- **文件**：`server/src/services/mysqlService.ts`
- **修复时间**：2026-03-30
- **修复内容**：将版本检查放在 UPDATE WHERE 条件中原子执行
  ```typescript
  const { error, count } = await supabase
    .from('tasks')
    .update({ ...fields, version: expectedVersion + 1 })
    .eq('id', id)
    .eq('version', expectedVersion)  // 原子版本检查
  if (count === 0) throw new Error('VERSION_MISMATCH')
  ```
- **状态**：✅ 已修复并验证

## 🟡 P2 — 性能优化（上线后持续改进）

### 6. TypeScript strict 模式未启用
- **文件**：`client/tsconfig.json` 第 14 行、`server/tsconfig.json` 第 7 行
- **现状**：两个配置文件均为 `"strict": false`
- **影响**：`null` / `undefined` 不安全访问无警告、any 类型推断宽松、潜在运行时错误未被捕获
- **建议**：上线后逐步修复，**不建议上线前强制开启**（会暴露大量错误，修复工程量大）
- **优先级**：低，上线后第一个技术债务排期

### 7. Zustand 未使用选择器（无选择器订阅）
- **文件**：`client/src/hooks/useStore.ts`
- **现状**：
  ```typescript
  // 当前：订阅整个 store
  const { tasks, milestones } = useStore()
  // 正确：只订阅需要的字段
  const tasks = useStore(state => state.tasks)
  ```
- **问题**：任何 store 字段变化（如 sidebarOpen）都会触发所有订阅组件重渲染
- **影响**：小数据量感知不到，数据量增大后页面卡顿
- **修复难度**：低（全文替换使用方式，约 20-30 处）

### 8. 页面统计计算未使用 useMemo
- **文件**：Dashboard 页面及各统计卡片组件
- **现状**：统计数据（任务完成率、风险分布等）在每次渲染时重新计算
- **问题**：数据量增大后每次页面交互（如打开 sidebar）都触发重计算
- **修复方案**：
  ```typescript
  const stats = useMemo(() => ({
    totalTasks: tasks.length,
    completedTasks: tasks.filter(t => t.status === 'completed').length,
    // ...
  }), [tasks])
  ```
- **修复难度**：低（在各统计卡片 return 前包一层 useMemo）
