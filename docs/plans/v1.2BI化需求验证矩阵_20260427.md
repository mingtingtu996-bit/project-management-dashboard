## 使用方式

- 每条需求有唯一编号（REQ-xx）
- `验证方式` 分为 `auto`（Claude 可自动验证）和 `manual`（需要你亲自操作）
- 执行队列中每个步骤关联需求编号，步骤完成后逐条跑验证
- 最终看通过率即可判断完成度
- 自动验证命令统一按当前 Windows 工作区可执行口径编写：优先使用 `rg`，文件存在性使用 `node -e "process.exit(require('fs').existsSync('path')?0:1)"`，避免使用不可用的 `grep / ls / head`
