# 展示层回调方案 V2｜P1 实施顺序与文件清单

文档定位：
- 本文档用于承接 `P0` 完成后的下一阶段展示层回调实施。
- 仅覆盖 `V2` 主线中的第二批支撑页、父模块承接页和报表承接收口。
- 只允许展示层改造，不允许改业务逻辑、数据口径、聚合规则、录入链路。

## P1 总原则

- 继续沿用 `展示层回调方案V2_前端实现任务拆分单_20260405.md`
- 先收父模块承接，再收支撑页，再收报表承接，再做隐藏入口最终收口
- `TaskManagement` 和 `LicenseManagement` 先把“父模块认知”彻底做成立，不新增重型总览页
- `Reports` 只做“按业务归属拆回模块”，不再把独立报表页做成主入口
- `Monitoring` 仍为隐藏工具路由，不进入主导航
- `TeamMembers` 仍按 Header / 项目辅助能力处理，不回主导航
