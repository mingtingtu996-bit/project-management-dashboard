## 执行方法补充

- 正文任一步骤开始前，必须先对照《三份专项方案执行前检查清单.md》筛出本步适用的 `blocked` 条件；若命中任一阻断项，当前步骤不得直接进入实现。
- 下列步骤已在正文内明确拆成子步骤：`15.1 / 15.3 / 15.4 / 15.5 / 15.6 / 15.7 / 15.8 / 15.9 / 15.10`；执行时不再依赖模型自行判断是否拆分。
- 子步骤只允许内联本步必需的需求摘录，推荐控制在 `20-80` 行，并同时保留来源锚点 `业务流程优化清单.md:xxxx`；不得全文复制需求章节，避免形成第二份需求真值。
- 数据模型 / API / 状态机类步骤统一采用 `test -> impl -> review`；页面壳层 / 组件类步骤统一采用 `contract-test -> impl -> review`；纯视觉一致性统一采用 `story/e2e -> review`。
- 高风险页面步骤的子步骤命名统一采用 `15.xa-contract-test -> 15.xa-impl -> 15.xa-review` 这一组模式；同一父步骤下的下一块功能继续用 `15.xb-* / 15.xc-*` 顺延，父步骤总复核统一写成 `15.x-review`。
- `review` 步骤只做检查，不写新功能；若发现缺口，当前父步骤必须保持 `blocked`，不得直接记为 `completed`。
- `review` 发现的问题写回 `EXECUTION_PROGRESS.json` 时，必须使用统一 `gaps` 结构，至少包含 `id / source / rule / finding / fix_in` 五个字段，例如：

```json
{
  "step": "15.3-review",
  "status": "blocked",
  "gaps": [
    {
      "id": "GAP-15.3-01",
      "source": "业务流程优化清单.md:5660",
      "rule": "确认冻结吸底条：存在阻断级问题时主按钮禁用",
      "finding": "BaselineConfirmBar 未检查 blockingIssues > 0",
      "fix_in": "15.3-fix-01"
    }
  ]
}
```

- 跨文件前置依赖必须显式声明；`15.0 planning-types` 的前置步骤固定为 `10.1 completed`、`10.2b completed`。三份专项方案的实际交错顺序应理解为：`10.1 → 10.2a → 10.2b → 15.0 → 15.0a → 15.1 ...`，而不是做完全部 `10.x` 后再进入 `15.x`。
- 计划编制的 E2E / story 验收不能只覆盖主流程；至少必须覆盖：`并发锁冲突 / 来源版本过期 / 数据不足降级态 / 关账处理中被他人先处理 / 快速确认路径 vs 标准路径`。

- **具体动作**
  1. 先定义计划编制域共享类型：`BaselineVersion / BaselineItem / MonthlyPlanVersion / MonthlyPlanItem / CarryoverItem / RevisionPoolCandidate / PlanningStatus / PlanningEvent`
  2. 明确共享 API 契约：至少锁定 `基线确认冻结 / 月计划确认 / 月末关账 / 独立修正 / 观察池读取与提交 / 偏差分析读取` 的 request / response / error shape

  4. 为状态机补专门测试：验证合法转换可通过、非法转换被拒绝，覆盖 `draft / confirmed / closed / revising / pending_realign` 等核心状态与事件组合

     ```json
     {
       "types": ["BaselineVersion", "MonthlyPlanVersion"],
       "endpoints": [
         {
           "method": "POST",
           "path": "/api/task-baselines/:id/confirm",
           "requestShape": "{ version: number }",
           "responseShape": "{ id: string, status: 'confirmed' }",
           "errorCodes": ["VERSION_CONFLICT", "BLOCKING_ISSUES_EXIST"]
         }
       ],
       "stateMachine": {
         "states": ["draft", "confirmed", "closed"],
         "events": ["CONFIRM", "CLOSE_MONTH"],
         "transitions": ["draft + CONFIRM -> confirmed"]
       }
     }
     ```
- **不要做的事**
  - 不在本步编写页面 UI
  - 不在本步落数据库迁移和业务路由副作用链
  - 不重复定义 `10.1 / 10.2b` 已落地的共享类型
