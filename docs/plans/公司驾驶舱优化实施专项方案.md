# 公司驾驶舱优化实施专项方案

编制日期：2026-04-15  
需求来源：`v2功能增强需求文档_20260414.md` §四

## 1. 方案目标

本专项用于把公司驾驶舱从“基础汇总看板”升级为“公司级项目监控与干预入口”，让管理层不进入具体项目，也能快速识别：

1. 哪些项目需要关注。
2. 哪些项目出现偏差、延期、阻碍、关账异常。
3. 哪些项目存在高优先级预警。
4. 当前用户在公司层面能看到哪些项目。

## 2. 当前现状与缺口

结合现有代码，当前公司驾驶舱已具备一定基础：

- 已有独立的 `CompanyCockpit` 页面与 `/api/dashboard/projects-summary` 数据链路。
- Hero 区已有 3 张摘要卡。
- 项目卡片已能展示总体进度、专项达成和若干快捷入口。
- Insight 区已具备图表区块结构。

但距离 v2 目标仍有明显差距：

- Hero 区缺少“需关注项目数”。
- 项目卡片缺少总工期偏差、活跃延期审批数、活跃阻碍数、月度关账状态。
- 快捷入口缺少“任务列表”。
- Insight 区仍保留信息密度偏低的项目矩阵图，没有收口成“项目信号排行 + 预警概览”。
- 全局角色过滤尚未真正落到公司驾驶舱数据查询。

## 3. 目标业务形态

## 3.1 Hero 区

从 3 张卡扩展为 4 张卡：

1. 项目总数
2. 平均进度
3. 平均健康度
4. 需关注项目数

其中“需关注项目数”定义为：

- 健康度 < 60，或
- 存在逾期里程碑

## 3.2 项目卡片

每个项目卡片除保留原有信息外，新增：

- 总工期偏差天数
- 活跃延期审批数
- 活跃阻碍数
- 本月关账状态
- “任务列表”快捷入口

## 3.3 Insight 区

图表区块保留 2 x 2 布局，但内容调整为：

1. 健康热力图：保留。
2. 里程碑达成图：补入偏移里程碑数。
3. 风险气泡图：对齐统一风险/问题/阻碍底座。
4. 项目信号排行：替换旧项目矩阵图，并合并跨项目预警概览职责。

## 3.4 权限过滤

公司驾驶舱所有数据统一按 `global_role` 过滤：

- `company_admin`：看全部项目。
- `regular`：只看本人参与项目。

## 4. 实施设计

## 4.1 数据接口扩展

建议对现有 `/api/dashboard/projects-summary` 做增量扩展，而不是新造多条碎片接口。

每个项目汇总对象建议补齐以下字段：

| 字段 | 说明 |
| --- | --- |
| `attention_required` | 是否属于需关注项目 |
| `schedule_variance_days` | 总工期偏差天数 |
| `active_delay_requests` | 活跃延期审批数 |
| `active_obstacles` | 活跃阻碍数 |
| `monthly_close_status` | 未开始 / 进行中 / 已完成 / 已超期 |
| `unread_warning_count` | 未读预警数 |
| `highest_warning_level` | 最高级别预警 |
| `highest_warning_summary` | 最高级别预警摘要 |
| `shifted_milestone_count` | 已偏移里程碑数 |
| `critical_path_affected_tasks` | 关键路径受影响任务数 |

同时返回 Hero 区聚合值：

- `project_count`
- `avg_progress`
- `avg_health_score`
- `attention_project_count`

## 4.2 后端聚合逻辑

建议将公司驾驶舱聚合拆分为 3 层：

1. `ProjectSummaryAggregator`
   - 单项目聚合总体进度、专项达成、偏差、阻碍、关账状态。

2. `CompanyDashboardAggregator`
   - 聚合 Hero 与 Insight 区数据。

3. `CompanyVisibilityScopeResolver`
   - 按 `global_role` 解析当前用户可见项目范围。

这样可以避免把项目过滤、项目聚合、图表聚合继续混在同一个路由函数里。

## 4.3 前端页面改造

### 4.3.1 Hero 区

布局从：

- `md:grid-cols-3`

调整为：

- `md:grid-cols-2 xl:grid-cols-4`

新增第四卡“需关注项目数”，采用红/琥珀语义色，突出管理动作感。

### 4.3.2 ProjectOverviewSection

每张项目卡新增三组信号：

1. 偏差信号
   - 显示总工期偏差天数。

2. 执行受阻信号
   - 活跃延期审批数。
   - 活跃阻碍数。

3. 关账信号
   - 本月关账状态。

快捷入口从 3 个增至 4 个：

- 进入项目
- 任务列表
- 风险与问题
- 里程碑

### 4.3.3 CompanyInsightSection

保留现有容器和视觉体系，但组件职责重组：

- `ProjectMatrixMap` 替换为 `ProjectSignalRanking`
- 把跨项目预警概览并入 `ProjectSignalRanking`

每行项目显示：

- 健康度
- 延期审批待处理数
- 关账超期天数
- 关键路径受影响任务数
- 活跃阻碍数
- 未读预警数
- 最高级别预警摘要

点击项目行支持跳转：

- 提醒中心
- 项目 Dashboard

## 4.4 与其他专项的依赖关系

公司驾驶舱优化不是孤立开发，依赖以下专项输出：

- 登录权限系统专项：
  - `global_role`
  - 公司级项目可见范围

- 数据质量专项：
  - 数据置信度
  - 项目级质量信号

- 责任主体专项：
  - Dashboard 责任单位视角收口

- 业务流程优化现有底座：
  - 偏差信号
  - 关账状态
  - 风险/问题/阻碍统一底座

## 5. 分阶段执行

## 5.1 第一阶段：接口扩展与可见范围收口

- `projects-summary` 扩字段。
- `global_role` 过滤接入。
- Hero 第四卡接入。

交付结果：

- 公司驾驶舱先具备“看见哪些项目”和“哪些项目要关注”的核心能力。

## 5.2 第二阶段：项目卡片增强

- 卡片补偏差信号、延期/阻碍信号、关账状态。
- 增加“任务列表”快捷入口。

交付结果：

- 管理层不用进入项目，也能判断执行态势。

## 5.3 第三阶段：Insight 区重组

- 里程碑达成图补偏移维度。
- 风险气泡图数据源对齐。
- 项目矩阵图替换为项目信号排行。

交付结果：

- 公司驾驶舱从“静态概览”升级为“可排序、可比较、可下钻”的监控台。

## 6. 涉及模块

后端：

- `server/src/routes/dashboard.ts`
- `server/src/services/projectExecutionSummaryService.ts`
- 项目级偏差、阻碍、关账、预警聚合服务
- 权限过滤与当前用户可见项目解析逻辑

前端：

- `client/src/pages/CompanyCockpit.tsx`
- `client/src/services/dashboardApi.ts`
- `client/src/components/company-cockpit/*`
- 导航与快捷跳转链接

## 7. 验收标准

1. Hero 区由 3 卡扩展为 4 卡，并新增“需关注项目数”。
2. 公司驾驶舱数据按 `global_role` 正确过滤。
3. 项目卡片可展示总工期偏差、活跃延期审批数、活跃阻碍数、关账状态。
4. 项目卡片新增“任务列表”快捷入口。
5. 项目信号排行替换旧项目矩阵图，并承接跨项目预警概览。
6. 里程碑达成图可展示偏移里程碑维度。
7. 点击项目行可跳转到对应项目提醒中心或 Dashboard。

## 8. 风险与注意事项

- 项目卡片信息密度显著提升，必须控制行高和层级，否则会破坏当前页面节奏。
- `projects-summary` 已承载多个页面，扩字段时要注意兼容 Dashboard、Reports 等现有消费者。
- 预警、偏差、关账三类信号刷新频率不同，聚合口径要明确“实时值”还是“最近一次快照”。
- `regular` 用户看不到全部项目时，Hero 聚合值必须同步按可见范围重算，不能显示全公司口径。

## 9. 建议优先级

- P1：`global_role` 过滤、Hero 第四卡。
- P2：项目卡片增强。
- P2：项目信号排行替换与预警概览合并。
