## 一、前期证照模块（9 PARTIAL + 1 MISSING）

### 1.1 Hover 交互联动

**问题 #1：Hover 弱化不完整**
- **现状**：看板卡片 hover 时，共享带未完全弱化非关联证件
- **期望**：hover 证件卡片时，共享带中非关联证件应降低透明度至 0.4
- **影响**：视觉引导不够清晰
- **文件**：`client/src/pages/PreMilestones/components/FourCertificateBoard.tsx`

**问题 #2：证件行反向 Hover 无联动**
- **现状**：hover 共享带中的工作项时，看板中关联证件卡片无高亮
- **期望**：hover 共享带工作项时，看板中关联的证件卡片应加边框高亮
- **影响**：反向追溯不直观
- **文件**：`client/src/pages/PreMilestones/components/CertificateSharedRibbon.tsx`

### 1.2 交互行为缺失

**问题 #3：阻塞标签无 scrollIntoView**
- **现状**：点击"阻塞标签"时无滚动定位
- **期望**：点击阻塞标签应滚动到台账中对应的阻塞工作项行
- **影响**：快速定位能力缺失
- **文件**：`client/src/pages/PreMilestones/components/FourCertificateBoard.tsx`

**问题 #4："预计开工日期"仅 toast 无 Dialog**
- **现状**：点击"预计开工日期"仅弹 toast 提示
- **期望**：应弹出 Dialog 显示详细的证件依赖关系和预计时间计算逻辑
- **影响**：用户无法查看详细推算依据
- **文件**：`client/src/pages/PreMilestones.tsx`

**问题 #5："逾期事项"卡片不可点击**
- **现状**：总览区"逾期事项"卡片无点击交互
- **期望**：点击应筛选台账显示所有逾期工作项
- **影响**：快速筛选能力缺失
- **文件**：`client/src/pages/PreMilestones/components/LifecycleSummaryCards.tsx`

**问题 #6：台账升级按钮 props 未传入**
- **现状**：台账行级"升级为问题/风险"按钮存在但 props 未传入，无法触发
- **期望**：传入 `onEscalateIssue` / `onEscalateRisk` 回调
- **影响**：无法从台账直接升级
- **文件**：`client/src/pages/PreMilestones/components/CertificateLedger.tsx`

### 1.3 视觉规范偏差

**问题 #7：依赖矩阵待办颜色应为蓝非灰**
- **现状**：依赖矩阵中 `status: 'pending'` 单元格显示为灰色（slate）
- **期望**：应显示为蓝色（blue-100 背景 + blue-700 文字）
- **影响**：与设计规范不一致
- **文件**：`client/src/pages/PreMilestones/components/CertificateDependencyMatrix.tsx`

### 1.4 移动端适配

**问题 #8：移动端未适配**
- **现状**：前期证照页面在移动端布局错乱
- **期望**：响应式布局，移动端看板改为单列，台账支持横向滚动
- **影响**：移动端不可用
- **文件**：`client/src/pages/PreMilestones.tsx` 及所有子组件

### 1.5 筛选状态同步（MISSING）

**问题 #9：看板筛选与台账筛选未同步**
- **现状**：看板和台账各自维护筛选状态，互不影响
- **期望**：看板筛选器（证件类型/状态）应同步到台账筛选，反之亦然
- **影响**：用户需要重复设置筛选条件
- **文件**：`client/src/pages/PreMilestones.tsx`
- **优先级**：⭐⭐⭐ 高（唯一 MISSING 项）

## 二、施工图纸模块（4 PARTIAL + 2 MISSING）

### 2.3 信息展示不完整

- **期望**：应显示结构化的影响对象列表（影响的任务/验收/证照，带链接跳转）
- **影响**：影响范围不清晰
- **文件**：`client/src/pages/Drawings/components/DrawingVersionDialog.tsx`

### 2.4 视觉规范偏差

**问题 #15：影响对象摘要标签未在卡片显示**

- **期望**：卡片底部应显示影响对象摘要标签
- **影响**：关键信息不可见
- **文件**：`client/src/pages/Drawings/components/DrawingPackageBoard.tsx`

**问题 #16：关键卡点专业文案未特化**
- **现状**：总览区"当前关键卡点专业"显示通用文案
- **期望**：应根据实际数据显示具体专业名称（如"结构专业施工图审查"）
- **影响**：信息不够具体
- **文件**：`client/src/pages/Drawings/components/DrawingReadinessSummary.tsx`

## 三、验收流程轴模块（6 PARTIAL）

### 3.1 拖拽功能受限

**问题 #17：流程板拖拽仅台账内可用**
- **现状**：仅在台账视图中可拖拽排序，流程板视图中不可拖拽
- **期望**：流程板视图中也应支持拖拽调整验收项位置和依赖关系
- **影响**：流程板交互受限
- **文件**：`client/src/pages/AcceptanceTimeline/components/AcceptanceFlowBoard.tsx`

**问题 #18：拖拽无 snap 动画**
- **现状**：拖拽释放后无吸附动画
- **期望**：拖拽释放后应有平滑的吸附动画
- **影响**：交互体验不够流畅
- **文件**：`client/src/pages/AcceptanceTimeline/components/AcceptanceFlowBoard.tsx`

### 3.2 批量操作缺失

**问题 #19：无批量操作功能**
- **现状**：无法批量修改验收项状态、责任单位等
- **期望**：台账应支持多选 + 批量操作（批量修改状态/责任单位/计划日期）
- **影响**：批量处理效率低
- **文件**：`client/src/pages/AcceptanceTimeline/components/AcceptanceLedger.tsx`

### 3.3 表单字段不完整

**问题 #20：新增验收项缺字段**
- **现状**：新增验收项弹窗缺少 `is_hard_prerequisite` / `responsibility_unit` / `category` 字段
- **期望**：新增弹窗应包含所有必要字段
- **影响**：新建后需二次编辑
- **文件**：`client/src/pages/AcceptanceTimeline.tsx` (AddPlanDialog)

### 3.4 状态推进校验缺失

**问题 #21：无状态推进校验**
- **现状**：可直接跳状态（如从"草稿"直接到"已通过"）
- **期望**：状态推进应校验前置状态和依赖条件
- **影响**：数据一致性风险
- **文件**：`client/src/pages/AcceptanceTimeline/components/AcceptanceDetailDrawer.tsx`

### 3.5 关联可视化不足

**问题 #22：图纸/证照关联可视化有限**
- **现状**：详情抽屉中仅列出关联的图纸/证照ID
- **期望**：应显示关联对象的状态卡片，支持点击跳转
- **影响**：关联关系不直观
- **文件**：`client/src/pages/AcceptanceTimeline/components/AcceptanceDetailDrawer.tsx`

## 四、关键路径模块（6 PARTIAL）

### 4.1 环检测告知缺失

**问题 #23：环检测静默无告知**
- **现状**：检测到依赖环时静默处理，用户无感知
- **期望**：检测到环时应弹窗告知用户，并标注环中的任务
- **影响**：用户不知道为何关键路径计算失败
- **文件**：`client/src/lib/criticalPath.ts`

### 4.2 手动干预入口受限

**问题 #24：行级手动干预仅在弹窗**
- **现状**：仅在关键路径详情弹窗中可手动标记关注/插链
- **期望**：甘特图任务行级右键菜单应支持"标记为关键路径关注"
- **影响**：操作路径过长
- **文件**：`client/src/pages/GanttView.tsx`

### 4.3 视觉规范微调

**问题 #25：弹窗尺寸与规范有偏差**
- **现状**：关键路径详情弹窗宽度为 900px
- **期望**：规范要求 1000px
- **影响**：视觉规范不一致
- **文件**：`client/src/components/CriticalPathGraph.tsx`

**问题 #26：节点尺寸与规范有偏差**
- **现状**：DAG 节点尺寸为 220×92px
- **期望**：规范要求 240×100px
- **影响**：视觉规范不一致
- **文件**：`client/src/lib/buildCriticalPathLayout.ts`

**问题 #27：边框色与规范有偏差**
- **现状**：手动链接边使用 `#f97316`（橙色）
- **期望**：规范要求 `#fb923c`（稍浅的橙色）
- **影响**：视觉规范不一致
- **文件**：`client/src/components/CriticalPathGraph.tsx`

### 4.4 风险提醒未降级

**问题 #28：riskAlert 未降级为只读**
- **现状**：`riskAlert.ts` 仍执行独立检测逻辑
- **期望**：应降级为只读显示辅助，不再独立判定关键任务
- **影响**：可能与后端快照不一致
- **文件**：`client/src/lib/riskAlert.ts`
