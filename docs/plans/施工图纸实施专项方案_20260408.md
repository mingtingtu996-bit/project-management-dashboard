## 0. 使用规则

### 0.1 适用前提与时序

- 本专项处理 `施工图纸`，默认属于 `证照管理` 父模块下的平级子页。
- 本专项不把图纸重新并回 `前期证照` 或 `验收时间轴` 页面。
- 本专项允许同时修改前端、后端、迁移、测试和文档。
- 本专项与 `v1` 第 10 章中的 `10.3-10.7` 对应，总方案保留 gate 与索引里程碑，详细执行以本专项为准。

### 0.2 范围边界

- 本专项只处理图纸域。
- 本专项允许与任务条件、验收前置做联动。
- 本专项不重做前期证照主视图。
- 本专项不重做验收流程轴主视图。

### 0.3 核心口径

- 施工图纸主页面不是关系图，而是 `图纸准备度总览 + 图纸包主视图 + 图纸台账 + 详情处理`。
- 图纸组织采用 `专业 + 用途/属性 + 图纸包` 三层。
- 页面主对象是 `图纸包`，不是单张图纸。
- 图纸缺漏、齐套、送审要求、当前有效版和变更是核心能力。
- 图纸问题判别采用 `规则自动识别 + 人工确认升级`。
- 页面必须同时支持 `状态记录` 与 `问题展示`：
  - 状态记录：图纸包状态、送审状态、当前有效版、版本变更记录
  - 问题展示：缺漏、未齐套、逾期送审、多轮退审、工期影响标记
- 视觉风格必须延续现有系统的 `PageHeader + Breadcrumb + shell-surface + slate/blue/white card` 语言，不自创单独图纸主题。

### 0.4 绝对不要做的事

- 不继续让 `PreMilestones` 长期承接图纸主视图。
- 不把 `ForceDirectedGraph` 设为图纸主视图。
- 不把所有图纸一上来平铺到首页。
- 不要求每张图纸都人工标注“影响前期证照 / 影响任务 / 影响验收”。
- 不把版本管理只留在一个自由文本 `version` 字段里。
