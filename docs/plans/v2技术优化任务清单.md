# v2技术优化任务清单

编制日期：2026-04-15  
需求来源：`v2功能增强需求文档_20260414.md` §六 + §八-§十六

## 1. 清单说明

本清单将以下内容合并为一份执行表：

- 提醒中心优化
- 用户体验与新手引导
- 性能与技术债务
- 无障碍
- 工程化与运维
- 前端健壮性
- 数据库与数据安全
- CI/CD 与部署
- 业务健壮性
- 监控与可观测性

使用原则：

- P1：立即执行，解决明显风险、错误暴露、数据一致性和基础体验问题。
- P2：本轮增强，补齐系统稳定性、监控和效率提升。
- P3：整理优化，提升可维护性和长期体验。

## 5. 推荐执行波次

## 5.1 第一波：风险止血

优先处理以下事项：

- 提醒中心入口收口
- 全局错误友好提示
- 任务删除级联清理
- GanttView 重复请求、selector、memo
- 迁移执行修复
- CI 门禁
- 索引、导出、乐观锁、未保存保护

目标：

- 先解决“会坏数据、会暴露错误、会拖慢核心页”的问题。

## 5.2 第二波：稳定性增强

优先处理以下事项：

- 提醒中心自动消除与催办
- 请求取消、休眠同步保护、定时任务重试
- 数据保留策略
- 前端错误上报

目标：

- 让系统从“可用”升级为“可稳态运行、可定位问题”。

## 5.3 第三波：整理与长期维护

优先处理以下事项：

- 无障碍剩余项
- CI Actions 版本升级

目标：

- 降低长期维护成本，并补完易被延后的体验项。

## 6. 完成定义

本清单执行完成后，应达到以下结果：

1. 核心页面不再暴露技术错误给最终用户。
2. 核心数据链路具备删除一致性、导出一致性和高频协作写链路冲突保护。
3. GanttView、图纸详情等高频页面的性能与稳定性明显改善。
4. 部署、迁移、测试、日志形成最小可运维闭环。
5. 提醒中心、无障碍、错误提示、离线提示等体验细节完成收口。
