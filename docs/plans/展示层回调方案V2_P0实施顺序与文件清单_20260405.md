# 展示层回调方案 V2｜P0 实施顺序与文件清单

文档定位：
- 本文件用于后续代码实现模型直接按顺序施工。
- 只覆盖 `P0` 范围。
- 只允许展示层改造，不允许改业务逻辑、数据口径、聚合规则、录入链路。

## P0 总原则

- 先收壳层与导航，再收共享样式，再改高频页。
- 先建立页面归属和入口秩序，再补信息层和图示能力。
- `CompanyCockpit`、`Dashboard` 均为只读聚合页，只能改展示承接，不能改聚合逻辑。
- 一旦实现动作涉及 `services`、`hooks`、`lib` 中的业务计算、接口、持久化逻辑，立即停止。
