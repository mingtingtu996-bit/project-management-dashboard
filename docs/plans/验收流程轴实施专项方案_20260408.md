## 0. 使用规则

### 0.1 适用前提与时序

- 本专项作为 `v1` 后续正式专项处理，建议编号使用 `13.0-13.8`。
- `10.7` 只负责 `证照管理` 三子页的信息架构收口；验收时间轴的实质重构以本专项为准。
- 本专项允许同时修改前端、后端、迁移、测试和文档。
- 默认顺序建议为 `12.8 completed -> 13.0`；若人工明确调整优先级，至少要求 `10.7 completed`，且当前不存在其他证照管理相关 `in_progress` 步骤。

### 0.2 范围边界

- 本专项只处理 `验收时间轴`。
- 本专项默认不与 `前期证照`、`施工图纸` 混做。
- 本专项允许和 `任务管理 / 风险 / 预警` 做联动，但不改写这些域的主模型。

### 0.3 核心口径

- 验收主视图不是 `ForceDirectedGraph`。
- 验收关系仍保留在图里表达，但只使用 `卡片 + 箭头 + 分流/合流`。
- 页面主结构固定为：`流程板视图 + 台账视图 + 详情抽屉`。
- 数据模型固定拆为：`目录 / 实例 / 依赖 / 条件 / 记录` 五层。
- 主状态固定为单值；`受阻 / 临期 / 逾期` 等作为叠加标记。
- 页面必须同时支持 `状态记录` 与 `问题展示`：
  - 状态记录：阶段状态、申报/验收/整改/归档记录、前置满足记录
  - 问题展示：受阻、临期、逾期、整改未闭环、可转风险/问题事项
- 视觉风格必须延续现有系统的 `PageHeader + Breadcrumb + shell-surface + slate/blue/white card` 语言，不另起实验型图谱主题。

### 0.4 绝对不要做的事

- 不要继续把 `type_id / type_name / acceptance_type / name` 混在一个字段层解决。
- 不要继续让 `position.x / position.y` 成为页面布局真值。
- 不要把大面积关系线、虚线语义、自由画布重新引回主视图。
- 不要把验收、图纸、证照三块重新混成一个页面。
- 不要在没有完成新模型兼容前直接删除旧 `acceptance_plans` 读写链路。

- 并入 `10.10` 后，验收流程轴主数据应统一挂入 `acceptanceTimeline` 命名状态分区；流程板、台账、详情抽屉共享同一份验收项/前置/记录主数据，节点展开态、局部缩放、当前抽屉焦点等瞬时 UI 状态继续留在页面本地。

并入 `v1` 后应体现：

- `10.7` 只负责三子页架构收口，不再承担验收重构本体
- 验收时间轴的实质重构放入 `13.0-13.8`
- `13.x` 的详细执行以本专项 `AT.0-AT.8` 为准，不在 `v1` 主文档中重复展开实现细节
