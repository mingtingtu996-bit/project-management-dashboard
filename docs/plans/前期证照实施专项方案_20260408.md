## 0. 使用规则

### 0.1 适用前提与时序

- 本专项处理 `前期证照`，默认属于 `证照管理` 父模块下的平级子页。
- 本专项允许先完成前期证照页本身，再在施工图纸专项确认后做彻底拆页收口。
- 本专项可以和 `预警 / 风险 / 问题` 联动，但不改写这些模块主模型。
- 若正式并入 `v1` 主线，建议编号使用 `12.0-12.8`。
- 默认顺序建议为 `11.6 completed -> 12.0`；若人工明确调整优先级，至少要求 `10.7 completed`，且当前不存在其他证照管理相关 `in_progress` 步骤。

### 0.2 范围边界

- 本专项只处理开工前四证。
- 本专项不负责施工图纸主视图设计。
- 本专项不负责验收时间轴主视图设计。
- 三块未来都留在 `证照管理` 父模块下，但业务页边界必须清楚。

### 0.3 核心口径

- 前期证照的核心不是关系图，而是 `四证推进看板 + 办理台账 + 开工准入总览`。
- 页面主任务是回答“何时具备开工条件”。
- `责任单位` 在本专项中不单独拆内部责任单位字段，页面主字段统一用 `审批部门 / 主管部门`。
- 与 `预警 / 风险 / 问题` 联动采用弱耦合。
- 页面必须同时支持 `状态记录` 与 `问题展示`：
  - 状态记录：证件状态、办理事项状态、最近变更、补正/阻塞记录
  - 问题展示：临期、逾期、补正、共享事项阻塞、可升级问题
- 视觉风格必须延续现有系统的 `PageHeader + Breadcrumb + shell-surface + slate/blue/white card` 语言，不单独再起一套证照风格。

### 0.4 绝对不要做的事

- 不继续让 `PreMilestones` 长期承接图纸主视图。
- 不把四证办理重新做成复杂关系图页。
- 不把前期证照和验收时间轴状态机混用。
- 不把审批部门、证件状态、阻塞原因继续散落在自由文本里不收口。
