## 数据完整性锚点问题解决

### 问题：开工条件升级需要完成时间锚点，但当前完成时间不强制填写

### 解决方案：采用方案B - 基于计划时间+状态组合

```
前置任务类条件：
- 当前置任务的 end_date < now 且 status != 'completed' 时，自动创建问题
- 不依赖 actual_end_date（因为不强制填写）

其他类型条件：
- 当条件的 target_date < now 且 is_satisfied = false 时，自动创建问题

必填字段要求：
- tasks.end_date: 必填（延期计算、前置条件判断依赖）
- task_conditions.target_date: 必填（条件过期判断依赖）
```

### 数据完整性保障机制

1. **事务性保障**：跨表操作必须在数据库事务中完成
2. **容错机制**：关键数据缺失时提供降级处理
3. **时间统一**：统一使用UTC时间，避免时区问题
4. **链路维护**：删除记录时保持关联追踪完整性
