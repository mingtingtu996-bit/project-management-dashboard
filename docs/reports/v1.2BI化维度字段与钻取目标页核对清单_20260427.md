# v1.2 BI化 Step 2.0 维度字段与钻取目标页核对清单

编制日期：2026-04-27  
对应步骤：`16.10`  
用途：为第二步的趋势分析、维度切片和钻取联动提供现状底稿。

## 1. 维度字典来源

- 统一维度入口：`GET /api/scope-dimensions?projectId=...`
- 已确认 `Dashboard.tsx` 直接消费该接口作为项目维度来源。
- 当前复用的维度字典以项目维度配置为准，主要包括：
  - `building`
  - `specialty`
  - `phase`
  - `region`
- 原则：
  - 页面侧只消费统一维度字典，不自行拼造新的维度枚举。
  - 如果某条业务链缺少独立维度字段，则在清单里标记为“依赖上游关联”，不要伪造切片。

## 2. 四链路底层字段核对

### 2.1 任务链

- 主要可用字段：`building_id`、`specialty_type`、`phase_id`
- 现状判断：任务链是四条链里维度标签最完整的一条，可作为首批切片来源。
- 备注：`region` 仍以项目维度字典为准，不把任务表当成唯一来源。

### 2.2 风险链

- 主要可用字段：`specialty_type`
- 现状判断：风险链当前可稳定承接专业维度；`building_id` / `phase_id` 未见稳定的页面级参数入口。
- 备注：风险/问题详情目前以 `riskId`、`issueId` 作为钻取主键，维度型切片后续再补。

### 2.3 材料链

- 主要可用字段：`participant_unit_id`、`specialty_type`
- 现状判断：材料链已形成单位 + 专业两层切片，页面也已支持基础筛选参数。
- 备注：材料链当前不承诺楼栋 / 阶段级切片。

### 2.4 验收链

- 主要可用字段：`building_id`、`phase_code`（后端）、`phase_id`（项目侧语义）
- 现状判断：验收计划 / 台账层已具备楼栋和阶段过滤能力，但 `AcceptanceTimeline` 页面暂未接入 URL query 参数。
- 备注：验收链页面改造前，先补页面参数解析，再做图表 / 列表联动。

## 3. 钻取目标页 URL 参数支持现状

| 页面 | 当前支持的参数 | 当前状态 | 备注 |
| --- | --- | --- | --- |
| `GanttView` | `view`、`scale`、`compare`、`baselineVersionId`、`highlight` | 已支持 | 已可按视图、时间尺度、比较模式和高亮任务切换 |
| `RiskManagement` | `stream` / `tab`、`riskId`、`issueId` | 已支持基础钻取 | 已能直达风险 / 问题详情，但未承接维度型切片参数 |
| `Materials` | `q`、`unit`、`specialty` | 已支持 | 已能按关键字、单位、专业筛选 |
| `AcceptanceTimeline` | 无 query 参数支持 | 未支持 | 需后续补 `buildingId` / `phaseCode` 解析 |

## 4. 需要补齐的路由参数改造项

- `RiskManagement`：补齐维度型 query 参数透传，至少要能承接后续 `Reports` 的筛选结果。
- `AcceptanceTimeline`：补齐 `buildingId` / `phaseCode` 的 query 参数解析与初始化。
- 后续钻取统一约束：
  - 图表点击只负责生成目标 URL，不在图表层做业务过滤。
  - 目标页负责读取 URL 参数并完成页面级筛选。

## 5. 结论

- `16.10` 已完成现状核对。
- `GanttView`、`RiskManagement`、`Materials` 已具备可用参数入口。
- `AcceptanceTimeline` 仍需补 URL 参数解析。
- 四链路维度字段已整理，可直接作为 `16.11` 到 `16.15` 的实现前置依据。
