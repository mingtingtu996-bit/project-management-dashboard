## 第六项：生产环境变量配置 ❌ **阻断项**

### 问题
`server/.env` 和 `client/.env` 中的 Supabase 凭据仍为占位符，**服务器启动后将立即因数据库连接失败退出**（`index.ts:76` `process.exit(1)`）：

```bash
# server/.env（当前）
SUPABASE_URL=https://your-project.supabase.co   ← 占位符
SUPABASE_SERVICE_KEY=your-service-role-key       ← 占位符
SUPABASE_ANON_KEY=your-anon-key                  ← 占位符
CORS_ORIGIN=${CORS_ORIGIN:-http://localhost:5173} ← 开发域名
NODE_ENV=development                             ← 需改为 production

# client/.env（当前）
VITE_SUPABASE_URL=https://your-project.supabase.co ← 占位符
VITE_SUPABASE_ANON_KEY=your-anon-key               ← 占位符
```

### 上线前必须完成的操作

**后端 (`server/.env`)：**

## 第七项：前端存储模式 ❌ **阻断项**

### 问题
当前 `client/.env.local` 中 `VITE_STORAGE_MODE=local`，**所有数据存储在浏览器 LocalStorage**，多用户无法共享，且部署后数据随浏览器清空消失。

生产环境必须切换为：
```bash
VITE_STORAGE_MODE=supabase
```

同时需要在 `client/.env.local` 中填入实际 Supabase 凭据（与第六项联动）。
