## 二、发现的问题汇总

### 🔴 严重问题 (Critical)

#### 1. 权限系统未正确连接
**文件**: `client/src/components/PermissionGuard.tsx` (第 31, 59 行)  
**文件**: `client/src/hooks/usePermissions.ts` (第 35 行)

**问题描述**:
- `PermissionGuard` 组件中 `currentRole` 被硬编码为 `'admin'`
- `RoleGuard` 组件同样硬编码角色
- `usePermissions` hook 在没有成员表查询的情况下默认返回 `'editor'`

**代码片段**:
```tsx
// PermissionGuard.tsx 第 31 行
const currentRole: PermissionLevel = 'admin' // TODO: 从 usePermissions 获取

// usePermissions.ts 第 33-35 行
// TODO: 从项目成员表中获取权限
// 暂时返回 editor 作为默认权限
return 'editor'
```

**影响**: 权限控制完全失效，所有用户都具有管理员权限。

**修复建议**:
```tsx
// 应该使用 usePermissions hook
const { permissionLevel } = usePermissions()
const currentRole = permissionLevel
```

#### 2. 版本冲突处理逻辑不完整
**文件**: `client/src/pages/GanttView.tsx` (第 228-236 行)

**问题描述**:
当任务更新检测到版本冲突时，代码只是简单地继续更新，没有正确处理冲突。

**代码片段**:
```tsx
if (updated) {
  updateTask(editingTask.id, { ...editingTask, ...taskData })
  toast({ title: "任务已更新" })
} else {
  // 版本冲突，直接更新 - 这里应该处理冲突
  updateTask(editingTask.id, { ...editingTask, ...taskData })
  toast({ title: "任务已更新" })
}
```

**修复建议**:
```tsx
if (updated) {
  updateTask(editingTask.id, { ...editingTask, ...taskData })
  toast({ title: "任务已更新" })
} else {
  // 显示冲突解决对话框
  setConflictData({ local: editingTask, incoming: taskData })
  setShowConflictDialog(true)
}
```

### 🟡 警告问题 (Warning)

#### 3. useEffect 依赖不完整
**文件**: `client/src/pages/ProjectList.tsx` (第 31-33 行)

**问题描述**:
`loadProjects` 函数在 useEffect 中被调用，但依赖数组为空，且 `loadProjects` 使用了 `setProjects` 但没有将其作为依赖。

**代码片段**:
```tsx
useEffect(() => {
  loadProjects()
}, []) // 缺少依赖

const loadProjects = () => {
  // 使用了 setProjects 但没有在依赖中声明
  setProjects(sorted)
}
```

**修复建议**:
```tsx
useEffect(() => {
  const loadProjects = () => {
    // ...
    setProjects(sorted)
  }
  loadProjects()
}, [setProjects]) // 添加依赖
```

---

#### 4. useCallback 依赖不完整
**文件**: `client/src/hooks/useKeyboardShortcuts.tsx` (第 38 行)

**问题描述**:
`handleKeyDown` useCallback 依赖了 `shortcuts` 数组，但每次渲染都会创建新的数组引用，导致 useCallback 失效。

**代码片段**:
```tsx
const handleKeyDown = useCallback((event: KeyboardEvent) => {
  // ...
  for (const shortcut of shortcuts) {
    // ...
  }
}, [shortcuts, enabled]) // shortcuts 每次渲染都变
```

**修复建议**:
使用 ref 存储 shortcuts 或使用稳定的引用。

---

#### 5. 内存泄漏风险 - 事件监听未清理
**文件**: `client/src/lib/storageService.ts` (第 137-140 行)

**问题描述**:
StorageService 在构造函数中注册了事件监听器，但没有提供清理方法。

**代码片段**:
```ts
if (typeof window !== 'undefined') {
  window.addEventListener('online', () => this.handleOnline())
  window.addEventListener('offline', () => this.handleOffline())
}
```

**修复建议**:
添加 `destroy()` 或 `cleanup()` 方法来移除事件监听器。

---

#### 6. 性能问题 - 全表加载+内存过滤
**文件**: `client/src/lib/storageService.ts` (第 193-228 行)

**问题描述**:
`getTasks`, `getRisks` 等方法每次都加载全部数据然后在内存中过滤，数据量大时性能差。

**代码片段**:
```ts
getTasks: async (projectId) => {
  const tasks = taskDb.getAll() // 加载所有任务
  return projectId ? tasks.filter(t => t.project_id === projectId) : tasks // 内存过滤
}
```

**建议**:
对于大量数据，应该使用 IndexedDB 或实现分页加载。

---

#### 7. 类型定义不一致
**文件**: `client/src/pages/GanttView.tsx` (第 31-51 行)

**问题描述**:
本地定义的 Task 类型与 `localDb.ts` 中的 Task 类型不一致，存在字段映射问题 (`title` vs `name`, `assignee` vs `assignee_name`)。

**代码片段**:
```tsx
interface Task {
  title?: string
  name?: string  // 兼容旧字段
  // ...
  assignee?: string
  assignee_name?: string
}
```

**建议**:
统一类型定义，避免字段冗余。

### 🟢 建议问题 (Suggestion)

#### 8. 缺少 useMemo 优化
**文件**: `client/src/pages/ProjectDetail.tsx` (第 81-86 行)

**问题描述**:
统计数据计算没有使用 useMemo，每次渲染都会重新计算。

**代码片段**:
```tsx
// 每次渲染都重新计算
const totalTasks = tasks.length
const completedTasks = tasks.filter((t) => t.status === 'completed').length
// ...
```

**修复建议**:
```tsx
const stats = useMemo(() => {
  const totalTasks = tasks.length
  const completedTasks = tasks.filter((t) => t.status === 'completed').length
  // ...
  return { totalTasks, completedTasks, /* ... */ }
}, [tasks, risks, milestones])
```

## 六、总结
