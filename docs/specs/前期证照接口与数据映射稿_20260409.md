# 前期证照接口与数据映射稿_20260409

> 适用范围：`证照管理 / 前期证照`
>
> 文档定位：承接 `前期证照字段模型+状态流转稿_20260409.md`，用于统一前后端接口、旧字段兼容和页面读写口径。
> 导航口径说明：侧边栏父级最终显示名统一为 `专项管理`；本文中保留的 `证照管理` 仅用于既有专项命名和技术语境。


## 1. 目标

本稿解决三件事：

- 现有 `pre-milestones` 旧链路如何兼容
- 新页面该读什么接口、写什么接口
- 旧字段如何映射到四证语义

## 2. 现有锚点

- `server/src/routes/pre-milestones.ts`
- `server/src/routes/pre-milestone-conditions.ts`
- `server/src/routes/pre-milestone-dependencies.ts`
- `server/src/routes/certificate-work-items.ts`
- `server/src/routes/certificate-dependencies.ts`
- `client/src/pages/PreMilestones.tsx`
- `client/src/pages/PreMilestones/types.ts`

## 3. 页面接口清单

建议页面主读接口：

- `GET /api/projects/:projectId/pre-milestones/board`
  - 返回总览、四证看板、共享事项条带摘要
- `GET /api/projects/:projectId/pre-milestones/ledger`
  - 返回办理台账
- `GET /api/projects/:projectId/pre-milestones/:certificateId/detail`
  - 返回证件详情、共享事项、依赖矩阵、状态记录

建议写接口：

- `POST /api/projects/:projectId/certificate-work-items`
- `PATCH /api/projects/:projectId/certificate-work-items/:id`
- `POST /api/projects/:projectId/certificate-dependencies`
- `DELETE /api/projects/:projectId/certificate-dependencies/:id`
- `POST /api/projects/:projectId/pre-milestones/:certificateId/escalate-issue`
- `POST /api/projects/:projectId/pre-milestones/:certificateId/escalate-risk`

## 4. 读模型

## 4.1 `board` 返回结构

```ts
interface LicenseBoardResponse {
  summary: {
    completedCount: number
    totalCount: number
    blockingCertificateType: string | null
    expectedReadyDate: string | null
    overdueCount: number
    supplementCount: number
    weeklyActionCount: number
  }
  certificates: CertificateBoardItem[]
  sharedItems: SharedRibbonItem[]
}
```

## 4.2 `ledger` 返回结构

```ts
interface LicenseLedgerResponse {
  items: CertificateWorkItemRow[]
  totals: {
    overdueCount: number
    blockedCount: number
    supplementCount: number
  }
}
```

## 4.3 `detail` 返回结构

```ts
interface LicenseDetailResponse {
  certificate: CertificateDetail
  workItems: CertificateWorkItemRow[]
  dependencies: DependencyMatrixCell[]
  records: StatusRecord[]
  linkedWarnings: LinkedWarningSummary[]
  linkedIssues: LinkedIssueSummary[]
  linkedRisks: LinkedRiskSummary[]
}
```

## 5. 旧字段到新字段映射

建议兼容映射：

- `milestone_type` -> `certificate_type`
- `milestone_name` -> `certificate_name`
- `issuing_authority` -> `approving_authority`
- `planned_date` -> `planned_finish_date`
- `completed_date` -> `actual_finish_date`
- `condition_status` -> `status`

旧库若没有明确四证类型时，映射规则建议：

- 中文名称包含 `土地` -> `land_certificate`
- 中文名称包含 `用地规划` -> `land_use_planning_permit`
- 中文名称包含 `工程规划` 或 `工规` -> `engineering_planning_permit`
- 中文名称包含 `施工许可` 或 `施工证` -> `construction_permit`

## 6. 兼容读写策略

读：

- 页面统一读 `board / ledger / detail`
- 服务端内部允许继续从 `pre_milestones` 旧表读取，再装配成新结构

写：

- 新增办理事项优先写 `certificate_work_items`
- 旧 `pre-milestones` 的更新接口可保留兼容，但页面不再直接依赖旧结构写 UI

删除或废弃：

- 不直接删除旧 `pre-milestone-conditions`、`pre-milestone-dependencies`
- 先在服务层映射，再逐步迁移

## 7. 页面组件与接口对应

- `LicenseReadinessSummary` -> `board.summary`
- `FourCertificateBoard` -> `board.certificates`
- `CertificateSharedRibbon` -> `board.sharedItems`
- `CertificateLedger` -> `ledger.items`
- `CertificateDetailDrawer` -> `detail`
- `CertificateDependencyMatrix` -> `detail.dependencies`

## 8. 升级与联动接口

建议一键升级接口返回统一结构：

```ts
interface EscalationResponse {
  success: boolean
  targetId: string
  targetType: 'issue' | 'risk'
  message: string
}
```

联动原则：

- 页面不直接改风险/问题主状态
- 只发起创建或关联动作

## 9. 给大模型的执行约束

- 如果接口命名与现有路由不一致，优先保证响应结构一致，再在服务端做别名兼容
- 页面层不允许直接消费旧 `milestone_*` 字段
- 所有页面组件都必须通过统一装配后的新响应结构读数据
