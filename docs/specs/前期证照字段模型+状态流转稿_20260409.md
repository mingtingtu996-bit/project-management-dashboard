# 前期证照字段模型+状态流转稿_20260409

> 适用范围：`证照管理 / 前期证照`
>
> 文档定位：承接 `前期证照产品改造稿_20260408.md` 与 `前期证照页面原型稿_20260409.md`，用于统一前期证照的数据对象、字段口径、状态流转和问题联动规则。

## 1. 文档目标

前期证照这块需要锁死三件事：

- 系统里到底有哪些核心对象
- 每个对象最少要存哪些字段
- 状态如何流转，什么情况下进入预警、问题、风险

本稿不追求做重模型，只服务当前已确认的四证推进页。

## 2. 对象模型

前期证照统一拆成三层：

1. `certificate_instance`
   四证中的一张证在某个项目中的实例
2. `certificate_work_item`
   办理事项，可被一个或多个证件共享
3. `certificate_dependency`
   证件与事项、事项与事项之间的依赖

补充一层记录对象：

4. `certificate_status_record`
   状态、补正、阻塞、推进记录

## 3. 对象定义

## 3.1 `certificate_instance`

用途：

- 承接页面上“四证四行”的主对象

建议核心字段：

- `id`
- `project_id`
- `certificate_type`
- `certificate_name`
- `status`
- `current_stage`
- `planned_finish_date`
- `actual_finish_date`
- `document_no`
- `approving_authority`
- `next_action`
- `next_action_due_date`
- `is_blocked`
- `block_reason`
- `latest_record_at`
- `created_at`
- `updated_at`

说明：

- `certificate_type` 只允许固定四类：
  - `land_certificate`
  - `land_use_planning_permit`
  - `engineering_planning_permit`
  - `construction_permit`

## 3.2 `certificate_work_item`

用途：

- 承接流程图里真实存在的办理事项
- 支撑共享事项条带和办理台账

建议核心字段：

- `id`
- `project_id`
- `item_code`
- `item_name`
- `item_stage`
- `status`
- `planned_finish_date`
- `actual_finish_date`
- `approving_authority`
- `is_shared`
- `next_action`
- `is_blocked`
- `block_reason`
- `created_at`
- `updated_at`

## 3.3 `certificate_dependency`

用途：

- 支撑共享事项和证件之间的阻塞/推进逻辑

建议核心字段：

- `id`
- `project_id`
- `predecessor_type`
- `predecessor_id`
- `successor_type`
- `successor_id`
- `dependency_kind`
- `notes`
- `created_at`

建议枚举：

- `predecessor_type / successor_type`
  - `certificate`
  - `work_item`
- `dependency_kind`
  - `hard`
  - `soft`

## 3.4 `certificate_status_record`

用途：

- 给“状态记录区”提供可读历史

建议核心字段：

- `id`
- `project_id`
- `target_type`
- `target_id`
- `record_type`
- `from_status`
- `to_status`
- `content`
- `recorded_at`
- `recorded_by`

建议枚举：

- `target_type`
  - `certificate`
  - `work_item`
- `record_type`
  - `status_change`
  - `supplement_required`
  - `blocked`
  - `unblocked`
  - `note`

## 4. 字段口径统一

页面主字段统一使用：

- `审批部门`

对应存储字段：

- `approving_authority`

页面常见显示字段与模型映射建议：

- `当前阶段` -> `current_stage` 或 `item_stage`
- `当前状态` -> `status`
- `下一动作` -> `next_action`
- `是否阻塞` -> `is_blocked`
- `阻塞原因` -> `block_reason`
- `最近更新` -> `latest_record_at` 或最近一条 `certificate_status_record.recorded_at`

## 5. 状态流转

## 5.1 证件主状态

建议固定为：

- `pending`
- `preparing_documents`
- `internal_review`
- `external_submission`
- `supplement_required`
- `approved`
- `issued`
- `expired`
- `voided`

页面中文映射：

- `待启动`
- `资料准备中`
- `报审中`
- `报批中`
- `待补正`
- `已批复`
- `已领证`
- `已失效`
- `已作废`

补充约定：

- 若底层英文枚举仍沿用 `internal_review / external_submission`，页面文案统一展示为 `报审中 / 报批中`
- 不允许前端页面、接口返回和文档口径并存 `内部报审 / 外部报批` 与 `报审中 / 报批中` 两套中文命名

## 5.2 合法流转

主链流转：

```text
待启动 -> 资料准备中 -> 报审中 -> 报批中 -> 已批复 -> 已领证
```

补正分支：

```text
报批中 -> 待补正 -> 报批中
已批复 -> 待补正 -> 报批中
```

终止分支：

```text
任意进行中状态 -> 已作废
已领证 -> 已失效
```

## 5.3 办理事项状态

建议固定为：

- `pending`
- `in_progress`
- `submitted`
- `supplement_required`
- `completed`
- `blocked`
- `cancelled`

页面中文映射：

- `待启动`
- `办理中`
- `已提交`
- `待补正`
- `已完成`
- `受阻`
- `已取消`

## 6. 阻塞与共享事项规则

规则 1：

- 一个 `hard` 依赖未满足时，对应证件 `is_blocked = true`

规则 2：

- 一个共享事项完成后，可同步刷新多个证件的“下一动作”和阻塞摘要

规则 3：

- 同一共享事项阻塞多个证件时，首页用一条共享事项条带表达，详情再展开矩阵

## 7. 预警 / 问题 / 风险联动

自动进入预警的建议条件：

- `next_action_due_date` 临近
- `planned_finish_date` 超期
- 同一对象多次进入 `supplement_required`
- 关键证件长期 `is_blocked = true`

建议升级为问题的条件：

- 某个具体 `certificate_work_item` 被退件或长期卡批

建议升级为风险的条件：

- 某张关键证件长期未推进且影响开工面

说明：

- 前期证照主状态不与问题/风险主状态混用
- 联动采用弱耦合
