# 验收流程轴接口与数据映射稿_20260409

> 适用范围：`证照管理 / 验收时间轴`
> 导航口径说明：侧边栏父级最终显示名统一为 `专项管理`；本文中保留的 `证照管理` 仅用于既有专项命名和技术语境。


## 1. 目标

本稿用于统一：

- 旧 `acceptance_plans` 如何映射到新五层模型
- 页面读写接口
- 流程板、台账、抽屉的数据装配结构

## 2. 现有锚点

- `server/src/routes/acceptance-plans.ts`
- `server/src/routes/acceptance-catalog.ts`
- `server/src/routes/acceptance-dependencies.ts`
- `server/src/routes/acceptance-requirements.ts`
- `server/src/routes/acceptance-records.ts`
- `client/src/services/acceptanceApi.ts`

## 3. 页面接口清单

建议主读接口：

- `GET /api/projects/:projectId/acceptance/board`
- `GET /api/projects/:projectId/acceptance/ledger`
- `GET /api/projects/:projectId/acceptance/:planId/detail`

建议写接口：

- `POST /api/projects/:projectId/acceptance/plans`
- `PATCH /api/projects/:projectId/acceptance/plans/:id`
- `POST /api/projects/:projectId/acceptance/dependencies`
- `DELETE /api/projects/:projectId/acceptance/dependencies/:id`
- `POST /api/projects/:projectId/acceptance/records`

## 4. 五层模型映射

- `acceptance_catalog`
  - 标准类型目录
- `acceptance_plan`
  - 项目中的实例节点
- `acceptance_dependency`
  - 前后依赖
- `acceptance_requirement`
  - 资料与前置条件
- `acceptance_record`
  - 申报、整改、通过等过程记录

## 5. 旧字段映射

- `type_id` -> `catalog_id`
- `type_name` -> `catalog_name`
- `acceptance_type` -> `catalog_code` 或兼容展示字段
- `name` / `acceptance_name` -> `acceptance_name`
- `planned_date` -> `planned_finish_date`
- `position` -> 仅保留为展示兼容字段，不作为业务真值
- `depends_on` -> `acceptance_dependency`

## 6. 读模型

## 6.1 `board`

```ts
interface AcceptanceBoardResponse {
  summary: {
    totalCount: number
    inProgressCount: number
    blockedCount: number
    overdueCount: number
  }
  cards: AcceptanceFlowCard[]
}
```

## 6.2 `ledger`

```ts
interface AcceptanceLedgerResponse {
  items: AcceptanceLedgerRow[]
}
```

## 6.3 `detail`

```ts
interface AcceptanceDetailResponse {
  plan: AcceptancePlanDetail
  dependencies: AcceptanceDependencyDetail[]
  requirements: AcceptanceRequirementDetail[]
  records: AcceptanceRecordDetail[]
  linkedTasks: LinkedTaskSummary[]
  linkedDrawings: LinkedDrawingSummary[]
  linkedCertificates: LinkedCertificateSummary[]
}
```

## 7. 兼容策略

读：

- 页面统一读 `board / ledger / detail`
- 服务端内部允许继续从 `acceptance_plans` 旧表装配

写：

- 新增依赖、记录、条件优先走新路由
- 旧 `acceptance-plans` 仍保留兼容读写窗口

## 8. 组件与接口对应

- `AcceptanceSummaryCards` -> `board.summary`
- `AcceptanceFlowBoard` -> `board.cards`
- `AcceptanceLedger` -> `ledger.items`
- `AcceptanceDetailDrawer` -> `detail`

## 9. 执行约束

- 页面层不允许再直接拿 `position` 当真值布局
- 前后端不允许继续各自发明 `type_id / acceptance_type / name` 的含义
