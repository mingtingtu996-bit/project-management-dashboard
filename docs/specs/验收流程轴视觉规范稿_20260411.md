> 适用范围：`证照管理 / 验收时间轴`
>
> 文档定位：作为 [证照管理三子页视觉统一规范稿_20260411.md](C:/Users/jjj64/WorkBuddy/20260318232610/证照管理三子页视觉统一规范稿_20260411.md) 的专项补充，用于统一验收流程轴页在流程板、泳道、关系线和流程卡片上的专属视觉规则，确保新页面与当前系统 `9.x` 收口后的 UI 规则保持一致。
> 导航口径说明：侧边栏父级最终显示名统一为 `专项管理`；本文中保留的 `证照管理` 仅用于既有专项命名和技术语境。


## 1. 总体视觉原则

- 页面视觉必须继续属于当前系统，而不是独立产品
- 保持白卡、浅边框、`slate + blue` 体系
- 复杂业务靠结构表达，不靠重色块和大面积装饰
- 流程板是复杂视图，但不是实验性画布

## 2. 页面壳规范

- 页面外层背景：
  - `bg-slate-50`
- 页面主容器：
  - 白底卡片或 `shell-surface`
  - 浅灰边框 `border-slate-200`
  - 圆角与当前 `Dashboard / Reports / PreMilestones` 保持一致
- `PageHeader`：
  - 只保留标题
  - 不恢复副标题和说明卡
- `Breadcrumb`：
  - 与证照管理其余子页保持同一位置和密度

## 3. 流程板容器规范

- 流程板外层：
  - 白底
  - 浅边框
  - 清晰留白
- 泳道背景：
  - 使用极浅色块或分隔线
  - 不使用高饱和整块底色
- 时间刻度：
  - 细线 + 小号 `slate` 文本
  - 不做粗重表格线

## 4. 卡片样式规范

- 验收项卡片：
  - `bg-white`
  - `border border-slate-200`
  - 轻阴影或无阴影
  - 圆角保持系统一致
- 卡片内容分层：
  - 标题：`text-slate-900`，中高权重
  - 次信息：`text-slate-500/600`
  - 状态信息：用 badge 表达，不用彩底整卡
- 卡片默认只显示：
  - 名称
  - 范围
  - 状态
  - 计划/实际日期摘要
  - 准备度或阻塞摘要

## 5. 状态色规范

- 主强调色：
  - 蓝色，用于主操作、当前选中和关键进度信息
- 状态色建议：
  - `待策划 / 准备中`：中性灰蓝
  - `待申报 / 已申报 / 验收中`：蓝色系
  - `整改中`：橙色系
  - `已通过 / 已归档`：绿色系
  - `阻塞 / 异常`：红色系
- 状态通过 badge 表达：
  - 轻底色 + 深文字
  - 不做高饱和纯色大块铺底

## 6. 关系线与结构表达规范

- 关系线只保留简洁箭头
- 分流 / 合流用结构和节点表达，不再使用虚线 / 实线双语义体系
- 默认线条颜色使用低对比 `slate`
- 选中卡片时再高亮上下游，不默认全图高亮
- 不允许回到力导向图那种“满屏连线”效果

## 7. 台账与抽屉规范

- 台账：
  - 延续系统标准表格和卡片密度
  - 不使用另起一套重数据平台风格
- 抽屉：
  - 白底
  - 清晰分区
  - 浅分隔线
  - 标题、按钮和输入框沿用现有 `Dialog / Drawer` 语言
- 抽屉中的操作按钮：
  - 主按钮蓝色
  - 次按钮白底边框
  - 危险动作只在必要位置出现

## 8. 视觉密度规范

- 默认只展开当前关键阶段，不一上来把全量关系和详情都摊开
- 复杂信息进抽屉，不堆在卡片正面
- 同屏优先保证：
  - 结构清楚
  - 时间可读
  - 状态可见
- 不追求一次性呈现所有业务字段

## 9. 响应式规范

- 桌面端：
  - 主视图为流程板
  - 台账与抽屉配合作为深层工作区
- 平板端：
  - 保留流程板，但压缩时间刻度和卡片宽度
- 移动端：
  - 默认进入台账
  - 流程板降级为横向阶段卡列
  - 抽屉改全屏弹层

## 10. 与 9.x UI 收口的一致性要求

- 继续遵守 `9.x` 已收口规则：
  - 无副标题
  - 无说明卡
  - 无大段引导文案
  - 统一 `card-base / shell-surface / border-slate-200 / bg-white`
- 不引入新的重色背景、复杂渐变和独立视觉主题

## 11. 禁止事项

- 不允许让流程板视觉回到 `ForceDirectedGraph` 风格
- 不允许使用复杂虚实线、彩色泳道和多套箭头图例
- 不允许把整页做成深色大屏风格
- 不允许为强调复杂性而牺牲系统一致性
