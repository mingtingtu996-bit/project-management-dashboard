> 适用范围：`证照管理 / 前期证照 / 施工图纸 / 验收时间轴`
>
> 文档定位：作为证照管理三子页的共享视觉规范，统一页壳、卡片、筛选、抽屉、状态标签和响应式语言；各专项若有更复杂视觉要求，只能在本规范基础上补充，不能另起一套视觉体系。
> 导航口径说明：侧边栏父级最终显示名统一为 `专项管理`；本文中保留的 `证照管理` 仅用于既有专项命名和技术语境。


## 1. 总体原则

- 三子页必须看起来属于同一个父模块
- 视觉语言继续服从当前系统 `9.x` 收口结果
- 用结构表达复杂度，不用额外说明卡堆信息
- 不让任一子页变成“独立产品风格”

## 2. 页面壳规范

- 页面背景统一：
  - `bg-slate-50`
- 主内容容器统一：
  - `shell-surface`
  - `bg-white`
  - `border border-slate-200`
  - 当前系统一致的圆角
- `PageHeader`：
  - 只保留标题
  - 不恢复副标题、说明卡、说明性引导段
- `Breadcrumb`：
  - 保持与项目其他主页面同一位置和密度

## 3. 导航与切换规范

- `证照管理` 作为父级导航，不在子页内部再造二级导航体系
- 子页切换按钮采用当前系统轻量 tab / segmented 风格
- 当前子页高亮使用蓝色系，不使用高饱和重底色
- 返回上级或切换同级页面时，优先保留当前筛选上下文

## 4. 卡片与面板规范

- 统一卡片语言：
  - `bg-white`
  - `border border-slate-200`
  - 轻阴影或无阴影
  - 统一圆角
- 首页卡片优先表达：
  - 当前状态
  - 关键数量
  - 关键日期
  - 阻塞摘要
- 不允许恢复 `card-v4 / card-v4-sm` 风格作为新设计基准

## 5. 筛选与工具栏规范

- 工具栏保持轻量，不堆积大段说明文字
- 筛选控件遵循：
  - 白底
  - 浅边框
  - 中性文字
  - 蓝色选中反馈
- 按钮优先级：
  - 主按钮：蓝底白字
  - 次按钮：白底边框
  - 危险动作：仅在必要处用红色文本或红框

## 6. 抽屉与弹窗规范

- 深层信息统一进入抽屉或弹窗
- 抽屉 / 弹窗统一：
  - 白底
  - 清晰分区
  - 浅分隔线
  - 标题明显，正文密度适中
- 不允许把复杂表单直接堆进首页卡片

## 7. 状态标签规范

- 状态尽量用 badge 表达，不用整卡彩底
- 建议色系：
  - 中性准备态：灰蓝
  - 进行中：蓝色
  - 待处理 / 整改 / 补正：橙色
  - 已完成 / 已通过：绿色
  - 阻塞 / 异常：红色
- 同一语义在三子页保持相近颜色，不允许同义不同色

## 8. 加载、空态、错误态规范

- 页面级加载优先用骨架屏
- 按钮级或提交级加载继续保留 spinner
- 空态只保留必要提示，不做说明性大卡
- 错误态优先 toast 或页内轻提示，不做重弹窗打断

## 9. 响应式规范

- 桌面端：
  - 保持“概览 + 主视图 + 深层抽屉”的三层结构
- 平板端：
  - 优先压缩工具栏与卡片密度
- 移动端：
  - 默认进入台账或列表视图
  - 抽屉改全屏弹层
  - 复杂主视图做降级，不强保桌面布局

## 10. 与专项视觉补充的关系

- 前期证照、施工图纸默认只使用本共享规范，不另起独立视觉规范稿
- 验收流程轴可在本规范基础上使用 [验收流程轴视觉规范稿_20260411.md](C:/Users/jjj64/WorkBuddy/20260318232610/验收流程轴视觉规范稿_20260411.md) 做流程板专项补充
- 若共享规范与专项补充发生冲突：
  - 页壳、标题、卡片、抽屉语言以共享规范为准
  - 专项图形表达细节以专项补充为准

## 11. 禁止事项

- 不允许恢复副标题、说明卡和大段引导文案
- 不允许三子页各自定义不同的卡片体系
- 不允许引入新的重主题色、深色大屏风格或复杂渐变体系
- 不允许为某一子页单独发明与系统不兼容的弹窗和工具栏语言
