## 4. 文件职责分工

- `产品改造稿`
  - 定业务边界和产品目标
- `字段模型/状态稿`
  - 定对象、字段和状态
- `页面原型稿`
  - 定页面结构、按钮和视觉层级
- `共享视觉规范稿`
  - 定三子页共用页壳、卡片、筛选、抽屉和状态语言
- `视觉规范稿`
  - 定专项补充视觉语言、卡片样式和状态色
- `接口与映射稿`
  - 定前后端读写和兼容策略
- `交互规则稿 / 模板配置稿`
  - 定页面行为或基础模板
- `测试与验收清单`
  - 定完成标准
- `实施专项方案`
  - 定文件落点、执行顺序和验收步骤

## 5. 大模型执行约束

- 不允许只看 `实施专项方案` 就直接编码
- 不允许只看 `页面原型稿` 就自行发明字段
- 不允许只看 `字段稿` 就自行发明页面布局
- 任何新增接口必须先对照 `接口与数据映射稿`
- 任何页面新增按钮必须先对照 `页面原型稿`
