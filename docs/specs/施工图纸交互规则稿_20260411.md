> 适用范围：`证照管理 / 施工图纸`
>
> 文档定位：用于统一施工图纸页在图纸包主视图、台账、版本窗口、送审规则触发和问题升级上的交互行为。
> 导航口径说明：侧边栏父级最终显示名统一为 `专项管理`；本文中保留的 `证照管理` 仅用于既有专项命名和技术语境。


## 1. 总体原则

- 首页先看图纸包结果，不先看单张图纸明细
- 图纸包是主对象，单张图纸和应有项是次级对象
- 当前有效版是默认主视角，历史版本统一进入版本窗口
- 自动识别先给结论，再允许人工确认，不做满屏告警堆叠

## 2. 页面级交互

- 顶部默认展示：
  - 图纸准备度总览
  - 图纸包主视图
  - 台账切换入口
- 点击 `新建图纸包`：
  - 打开图纸包弹窗
- 点击 `仅看缺漏`：
  - 主视图和台账同步切到缺漏视角
- 点击 `仅看送审`：
  - 主视图和台账同步切到送审视角
- 点击 `返回前期证照`：
  - 保留当前筛选条件后跳转

## 3. 图纸包主视图交互

- 点击图纸包卡片：
  - 打开 `DrawingPackageDrawer`
- 点击卡片上的 `查看缺漏`：
  - 抽屉定位到应有项缺漏区
- 点击卡片上的 `查看送审`：
  - 抽屉定位到送审状态区
- 点击卡片上的 `查看版本`：
  - 打开 `DrawingVersionDialog`
- Hover 图纸包卡片：
  - 高亮该包的状态标签和准备度条

## 4. 图纸包展开与收起

- 桌面端默认展开：
  - 当前筛选命中的图纸包
  - 存在缺漏、送审阻塞或变更中的图纸包
- 其余图纸包默认折叠
- 移动端默认全部折叠，点击后进入全屏详情
- 折叠状态只在当前页面会话内保留，不写入后端真值

## 5. 台账交互

- 台账默认按：
  - 图纸包状态
  - 计划日期
  - 专业
  的顺序排序
- 点击 `补录图纸`：
  - 打开单张图纸录入弹窗
- 点击 `查看详情`：
  - 打开图纸包或单张图纸详情抽屉
- 点击 `设为当前有效版`：
  - 弹出确认
  - 成功后刷新图纸包状态、版本窗口和应有项绑定

## 6. 版本窗口交互

- 版本窗口固定展示三层信息：
  - 当前有效版
  - 历史版本链
  - 变更说明
- 点击历史版本：
  - 右侧显示该版本的基本信息、替代关系和变更说明
- `设为当前有效版` 只允许在版本窗口或详情抽屉中触发
- 历史版本默认只读，不允许直接覆盖编辑

## 7. 送审规则交互

- `requires_review / review_mode` 默认来自图纸包模板
- 进入项目后允许项目级人工修正
- 当图纸包被标记为：
  - `必审`
  - `人工确认`
  时，主视图和抽屉显示送审标签
- 点击送审标签：
  - 抽屉定位到送审状态与规则依据区
- 送审规则变化后：
  - 同步刷新图纸包状态
  - 同步刷新缺漏/齐套结论

## 8. 自动识别与升级交互

- 系统自动识别以下问题：
  - 必需应有项缺漏
  - 无当前有效版
  - 必审图纸包未送审或未通过
  - 当前有效版变更后触发 `schedule_impact_flag`
- 自动识别结果在主视图只显示摘要标签，不直接展开长列表
- 点击 `升级为问题`：
  - 调用共享底座 `/api/issues`
  - 成功后卡片和抽屉显示“已关联问题”标记
- 点击 `升级为风险`：
  - 若共享底座支持，则进入风险创建链
  - 若当前会话尚未完成共享底座前置，则隐藏或禁用入口

## 9. 筛选联动规则

- 筛选项固定为：
  - `按专业`
  - `按图纸包状态`
  - `仅看缺漏`
  - `仅看送审`
  - `仅看变更`
- 主视图、台账、抽屉筛选保持同步
- 打开抽屉后切换筛选：
  - 抽屉不自动关闭
  - 当前对象若被筛掉，抽屉顶部显示“当前对象不在筛选结果中”

## 10. 移动端规则

- 默认进入台账视图
- 图纸包主视图降级为纵向卡片流
- 版本窗口改全屏弹层
- 详情抽屉改全屏详情页

## 11. 禁止事项

- 不允许把单张图纸平铺成首页主视图
- 不允许点击版本后直接隐式切换当前有效版
- 不允许把送审规则做成满屏流程图
- 不允许自动识别问题后不提供人工确认入口
- 不允许在本页私建独立问题真值，不接共享底座
