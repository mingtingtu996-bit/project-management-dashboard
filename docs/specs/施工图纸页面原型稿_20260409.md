# 施工图纸页面原型稿_20260409

> 适用范围：`证照管理 / 施工图纸`
>
> 文档定位：承接 `施工图纸产品改造稿_20260408.md`，用于统一页面布局、按钮位置、视觉层级和从当前双 tab 页迁出的方式。
> 导航口径说明：侧边栏父级最终显示名统一为 `专项管理`；本文中保留的 `证照管理` 仅用于既有专项命名和技术语境。


## 1. 页面目标

施工图纸页要回答的是：

- 当前哪些图纸包已具备施工条件
- 哪些图纸包存在缺漏或未齐套
- 哪些图纸包需要送审、正在送审、已退审
- 当前有效版本是哪一版
- 哪些图纸问题正在影响任务推进或验收前置

因此页面核心结构应是：

1. 图纸准备度总览
2. 图纸包主视图
3. 图纸台账
4. 详情抽屉
5. 版本变更窗口

## 2. 与当前页面的迁移关系

当前 `PreMilestones` 页面里存在 `施工图纸` tab。

目标状态必须明确为：

- `施工图纸` 从 `PreMilestones` 内部 tab 迁出
- 新增平级子路由 `/projects/:id/drawings`
- `Sidebar` 的 `证照管理` 下新增 `施工图纸`
- `PreMilestones` 中如需保留图纸入口，只允许保留摘要跳转，不再保留完整 tab 主体

结论：

- 后续实现时，不允许让 `PreMilestones` 和 `Drawings` 长期双主视图并存

## 3. 页面信息架构

## 3.1 页面骨架

```text
┌──────────────────────────────────────────────────────────────┐
│ Breadcrumb / 证照管理 / 施工图纸                            │
│ 施工图纸                              [返回前期证照]        │
│ 图纸准备度、送审状态、版本变更与任务/验收联动总览           │
├──────────────────────────────────────────────────────────────┤
│ [图纸包总数] [缺漏包] [必审包] [送审中] [工期影响]          │
├──────────────────────────────────────────────────────────────┤
│ 专业筛选 | 用途筛选 | 仅看缺漏 | 仅看送审 | 新建图纸包      │
├──────────────────────────────────────────────────────────────┤
│ 图纸包主视图                                                 │
│ 建筑 | 结构 | 机电 | 装修 ...                               │
│ 每个包显示：齐套度 / 当前有效版 / 送审 / 变更 / 影响标签     │
├──────────────────────────────────────────────────────────────┤
│ 图纸台账                                                     │
└──────────────────────────────────────────────────────────────┘
```

## 3.2 顶部总览

顶部摘要建议展示：

- 图纸包总数
- 缺漏图纸包数
- 必审图纸包数
- 送审中图纸包数
- 已出图可施工图纸包数
- 工期影响标记数

卡片下允许展示一行辅助说明，例如：

- `当前卡点：结构专业施工图审查`
- `竣工图仍缺 2 个应有项`

## 3.3 图纸包主视图

主视图先按专业分组，再在专业下看图纸包。

每个图纸包卡片只展示高频信息：

- 图纸包名称
- 专业
- 用途/属性
- 齐套度
- 当前有效版
- 送审要求
- 当前送审状态
- 是否有变更
- 是否影响工期
- 影响标签摘要

卡片不直接堆：

- 全量应有项
- 历史版本全链
- 长备注

## 4. 图纸台账

台账是明细查看和轻编辑入口。

推荐列：

- 图纸包
- 图纸名称 / 编号
- 当前版本
- 当前状态
- 是否当前有效版
- 送审要求
- 送审状态
- 是否变更
- 是否影响工期
- 操作

推荐操作按钮：

- `补录图纸`
- `查看详情`
- `查看版本`
- `设为当前有效版`

## 5. 详情抽屉

详情抽屉统一承接包级和单图级信息。

建议分 5 个区块：

1. 基本信息
2. 应有项与齐套
3. 送审状态
4. 状态记录
5. 影响与问题

### 5.1 基本信息

展示：

- 图纸包
- 图纸名称 / 编号
- 专业
- 用途/属性
- 当前状态
- 当前版本
- 是否当前有效版

### 5.2 应有项与齐套

展示：

- 应有项清单
- 已满足数
- 缺漏数
- 齐套度

操作按钮：

- `查看缺漏`

### 5.3 送审状态

展示：

- 是否必审
- 判定依据
- 当前送审状态
- 最近退审 / 通过记录

操作按钮：

- `查看送审`

### 5.4 状态记录

展示：

- 出图记录
- 版本变更记录
- 送审记录
- 最近更新时间

### 5.5 影响与问题

展示：

- 是否影响任务
- 是否影响验收
- 是否影响工期
- 自动识别的问题摘要

操作按钮：

- `升级为问题`
- `升级为风险`

## 6. 版本变更窗口

版本窗口独立于普通详情抽屉。

展示：

- 当前有效版
- 历史版本链
- 替代关系
- 变更说明

推荐按钮：

- `设为当前有效版`
- `查看历史版本`

## 7. 视觉统一要求

- 页面壳统一使用当前系统 `PageHeader + Breadcrumb + shell-surface`
- 顶部摘要卡延续现有系统白底卡片语言
- 图纸包卡片保持白底、浅边框、浅标签，不使用复杂图谱视觉
- 版本窗口和详情抽屉保持现有 `Dialog / Drawer` 风格
- 页面主层先展示“图纸包状态”，深层再看“单图和版本”，避免首页信息过载

## 8. 按钮清单

- 页面级按钮
  - `返回前期证照`
  - `新建图纸包`
  - `仅看缺漏`
  - `仅看送审`
- 卡片或台账按钮
  - `补录图纸`
  - `查看详情`
  - `查看版本`
  - `设为当前有效版`
  - `查看缺漏`
  - `查看送审`
  - `升级为问题`
  - `升级为风险`

## 9. 移动端降级

- 默认进入图纸台账
- 图纸包主视图改为纵向卡片列表
- 版本窗口改全屏弹层
- 详情抽屉改全屏详情页
