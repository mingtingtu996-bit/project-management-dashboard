# 施工图纸字段模型+状态规则稿_20260409

> 适用范围：`证照管理 / 施工图纸`
>
> 文档定位：承接 `施工图纸产品改造稿_20260408.md` 与 `施工图纸页面原型稿_20260409.md`，用于统一施工图纸的数据对象、字段口径、状态规则和自动判定逻辑。

## 1. 文档目标

施工图纸这块需要锁死四件事：

- 页面主对象到底是图纸包还是单张图
- 缺漏和齐套怎么判断
- 当前有效版和历史版本链怎么表达
- 送审、可施工、工期影响这些规则怎么判

本稿按当前已确认的最小口径来定，不引入复杂规则引擎。

## 2. 对象模型

施工图纸统一拆成四层：

1. `drawing_package`
2. `drawing_package_item`
3. `construction_drawing`
4. `drawing_version_record`

## 4. 字段口径统一

页面主对象统一为：

- `drawing_package`

页面关键显示字段与模型映射：

- `图纸包` -> `package_name`
- `专业` -> `discipline_type`
- `用途/属性` -> `document_purpose`
- `齐套度` -> `completeness_ratio`
- `缺漏数` -> `missing_required_count`
- `当前有效版` -> `current_version_drawing_id` 或 `is_current_version = true`
- `送审要求` -> `review_mode`
- `送审状态` -> `review_status`
- `是否有变更` -> `has_change`
- `是否影响工期` -> `schedule_impact_flag`

- `待启动`
- `编制中`
- `送审中`

- `已出图`
- `已完成`

## 5.2 单张图纸状态

## 6. 自动判定规则

## 6.1 缺漏判定

以下任一满足即判定为缺漏：

- `drawing_package_item.is_required = true` 且 `current_drawing_id` 为空
- 包内无任何当前有效版图纸，但模板要求至少存在一项必需图纸

## 6.2 齐套判定

齐套度计算方式：

```text
已满足的必需项数 / 应有必需项总数
```

齐套完成条件：

- 所有必需项均有当前有效版

## 6.3 送审要求判定

最小规则只基于：

- `discipline_type`
- `document_purpose`
- `package_code / package_name`

建议结果：

- `mandatory`
- `none`
- `manual_confirm`

## 6.4 当前有效版规则

规则：

- 同一张图纸编码在同一时间只能有一个 `is_current_version = true`
- 切换当前有效版时，上一版自动变为历史版

## 6.5 是否可施工

以下同时满足时为 `true`：

- 图纸包无必需项缺漏
- 当前有效版存在
- 若 `review_mode = mandatory`，则 `review_status = passed`

## 6.6 竣工图是否可验收

以下同时满足时为 `true`：

- `document_purpose = completion_archive`
- 图纸包无必需项缺漏
- 当前有效版存在
- 竣工归档类图纸包状态为 `issued` 或 `completed`

## 6.7 是否影响工期

满足以下任一条件时，`schedule_impact_flag = true`：

- 当前有效版发生变更，且该包已映射进行中的任务
- 必需项出现缺漏，且该包已映射进行中的任务
- 必审图纸包处于 `rejected` 或长期 `submitted`，且该包已映射进行中的任务

说明：

- 自动判定后允许人工修正
- 不做复杂智能判断

## 7. 问题展示与联动

页面要能自动识别这些问题：

- 缺漏
- 未齐套
- 必审未送审
- 送审超期
- 多轮退审
- 当前有效版缺失
- 工期影响标记

与其他模块的联动方式：

- 施工执行类图纸包 -> 任务条件
- 竣工归档类图纸包 -> 验收前置
- 自动识别的问题 -> 可升级为问题 / 风险
