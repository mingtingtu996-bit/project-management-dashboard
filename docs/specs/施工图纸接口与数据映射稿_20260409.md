# 施工图纸接口与数据映射稿_20260409

> 适用范围：`证照管理 / 施工图纸`
> 导航口径说明：侧边栏父级最终显示名统一为 `专项管理`；本文中保留的 `证照管理` 仅用于既有专项命名和技术语境。


## 1. 目标

本稿用于统一：

- 旧 `construction_drawings` 到新图纸包模型的映射
- 页面读写接口
- 图纸包、应有项、版本窗口的数据装配方式

## 2. 现有锚点

- `server/src/routes/construction-drawings.ts`
- `server/src/routes/drawing-packages.ts`
- `server/src/routes/drawing-review-rules.ts`
- `client/src/pages/Drawings.tsx`

## 3. 页面接口清单

建议主读接口：

- `GET /api/projects/:projectId/drawing-packages/board`
- `GET /api/projects/:projectId/drawing-packages/ledger`
- `GET /api/projects/:projectId/drawing-packages/:packageId/detail`
- `GET /api/projects/:projectId/drawing-packages/:packageId/versions`

建议写接口：

- `POST /api/projects/:projectId/drawing-packages`
- `PATCH /api/projects/:projectId/drawing-packages/:id`
- `POST /api/projects/:projectId/construction-drawings`
- `PATCH /api/projects/:projectId/construction-drawings/:id`
- `POST /api/projects/:projectId/drawing-packages/:packageId/set-current-version`

## 4. 读模型

## 4.1 `board`

```ts
interface DrawingsBoardResponse {
  summary: {
    totalPackages: number
    missingPackages: number
    mandatoryReviewPackages: number
    reviewingPackages: number
    scheduleImpactCount: number
  }
  packages: DrawingPackageCard[]
}
```

## 4.2 `ledger`

```ts
interface DrawingsLedgerResponse {
  drawings: DrawingLedgerRow[]
}
```

## 4.3 `detail`

```ts
interface DrawingDetailResponse {
  package: DrawingPackageDetail
  requiredItems: DrawingPackageItem[]
  drawings: DrawingLedgerRow[]
  records: VersionRecord[]
  linkedTasks: LinkedTaskSummary[]
  linkedAcceptance: LinkedAcceptanceSummary[]
}
```

## 5. 旧字段到新字段映射

- `drawing_type` -> `document_purpose`
- `version` -> `version_no`
- `drawing_status` -> `drawing_status`
- `review_status` -> `review_status`
- `planned_submit_date` -> `planned_submit_date`
- `planned_pass_date` -> `planned_pass_date`

新增包级字段：

- `package_code`
- `package_name`
- `discipline_type`
- `review_mode`
- `review_basis`
- `completeness_ratio`
- `missing_required_count`

## 6. 兼容策略

读：

- 页面统一读 `drawing-packages` 新结构
- 服务端内部允许继续从 `construction_drawings` 旧表装配

写：

- 新包、应有项、版本切换优先写新路由
- 单图 CRUD 可继续走旧 `construction-drawings`，但页面统一通过新 detail 结构回读

## 7. 组件与接口对应

- `DrawingReadinessSummary` -> `board.summary`
- `DrawingPackageBoard` -> `board.packages`
- `DrawingLedger` -> `ledger.drawings`
- `DrawingDetailDrawer` -> `detail`
- `DrawingVersionDialog` -> `versions`

## 8. 执行约束

- 页面层不允许直接拿旧 `construction_drawings` 原始行拼首页
- 首页只消费装配后的包级结构
- 版本窗口不允许依赖自由文本版号自行推断版本链
