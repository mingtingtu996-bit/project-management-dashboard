# Docs Layout

This directory collects non-code project materials that previously lived in the repo root.

- `reports/`: audit reports, test reports, verification notes, postmortems
- `plans/`: implementation plans, execution checklists, backlog and rollout notes
- `specs/`: product rules, field models, interface mappings, visual specs, prototypes
- `archive/`: uncategorized historical materials kept for reference

Root-level docs intentionally kept outside this folder:

- `README.md`
- `PROJECT_CONTEXT.md`
- `PROJECT_STATUS.md`
- `CLAUDE.md`
- `AUTO_MEMORY_CONFIG.md`
- `ENVIRONMENT_CONFIG.md`
- `TEST_RECORD.md`
