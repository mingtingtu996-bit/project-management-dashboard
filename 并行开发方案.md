## ⚠️ 重要注意事项

### 技术注意事项
1. **数据一致性风险**：必须实现乐观锁机制
2. **实时同步延迟**：设置2秒超时，离线缓存策略
3. **权限验证完整性**：每个API端点必须有权限检查
4. **性能监控**：实时监控API响应时间和内存占用

### 团队协作注意事项
1. **代码冲突预防**：每2小时合并一次代码
2. **沟通机制**：每日站会 + 进度同步 + 即时沟通
3. **版本控制**：明确分支策略和合并流程
4. **文档同步**：API文档、数据模型必须实时更新

### 风险管理注意事项
**高风险项**：
- Supabase集成失败（备选：保留local模式）
- 实时同步不稳定（降级：轮询模式）
- 权限系统漏洞（措施：严格测试 + 审计日志）
- 性能问题（措施：性能监控 + 告警）

**中风险项**：
- 数据导入失败
- 报表计算错误
- 移动端兼容性问题
- 部署配置错误

## 📊 审查工具和流程

### 自动化审查工具
```bash
# 代码质量
npm run lint          # ESLint检查
npm run type-check    # TypeScript类型检查
npm run test:coverage # 测试覆盖率

# 性能测试
npm run test:performance  # 性能基准测试
npm run bundle-analyze    # Bundle分析

# 安全扫描
npm run security:scan     # 安全漏洞扫描
npm run dependency:check  # 依赖安全检查
```

### 手动审查清单
1. **架构设计**：是否符合解耦原则
2. **代码质量**：是否有重复代码、复杂函数
3. **安全合规**：权限验证、输入验证
4. **性能指标**：响应时间、内存占用
5. **用户体验**：操作流畅性、错误处理

### 审查流程（每次30分钟）
1. **自动化检查**（15分钟）- 运行工具，收集报告
2. **功能演示**（10分钟）- 演示核心功能
3. **代码审查**（15分钟）- 关键代码逻辑审查
4. **问题整理**（5分钟）- 整理问题清单
5. **审查结论**（5分钟）- 通过/有条件通过/不通过
