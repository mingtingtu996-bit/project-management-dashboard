# 展示层回调方案 V2｜前端实现任务拆分单

使用方式：
- 本文件仅供前端实现模型逐页执行。
- 所有任务只允许落在展示层。
- 一旦实现动作触及数据口径、聚合逻辑、录入链路、业务契约，立即停止并回退到方案确认。

## 全局补充实施规则

### 页面承接关系表

| 旧入口 | 新归属 | 是否隐藏 | 是否保留路由 | 是否直接跳转 | 用户默认进入哪里 |
| --- | --- | --- | --- | --- | --- |
| `ProjectList` | `CompanyCockpit` | 是 | 是 | 是 | `CompanyCockpit` |
| 全局 `Dashboard` | `CompanyCockpit / 项目 Dashboard` | 是 | 是 | 是 | 公司级默认进 `CompanyCockpit` |
| `Reports` | 拆回各模块承接 | 是 | 是 | 否 | 项目默认进 `Dashboard`，统计入口按模块进入 |
| `Monitoring` | 隐藏工具路由 | 是 | 是 | 否 | 无默认进入，仅隐藏入口访问 |
| `TeamMembers` | `Header / 项目辅助面板` | 是 | 是 | 否 | 项目内当前页的辅助入口 |
| `GanttView` | `任务管理 / 任务列表页（历史路由名）` | 否 | 是 | 否 | `任务管理 / 任务列表` |
| `TaskSummary` | `任务管理` 子页 | 否 | 是 | 否 | `任务管理 / 任务总结` |
| `PreMilestones` | `证照管理` 子页 | 否 | 是 | 否 | `证照管理 / 前期证照` |
| `AcceptanceTimeline` | `证照管理` 子页 | 否 | 是 | 否 | `证照管理 / 验收时间轴` |

### 隐藏入口兼容策略

- 隐藏但保留路由：入口不在主导航显示，但历史路径仍可直接访问
- 隐藏且直接跳转：历史入口访问后直接跳到新归属页，不保留中间独立页面
- 隐藏且辅助承接：历史独立页面转为 Header、面板、抽屉等辅助入口，不再占主导航
- 所有隐藏策略都不得演变成删除页面、删除路由或切断原有高频入口

### 统一入口命名规范

- 公司级入口只使用：`公司驾驶舱`、`提醒中心`
- 项目级主模块只使用：`Dashboard`、`里程碑`、`WBS模板`、`任务管理`、`风险与问题`、`证照管理`
- 任务相关子页只使用：`任务列表`、`任务总结`
- `WBS` 只作为 `任务列表` 页内部结构表达，不作为独立子页命名
- 证照相关子页只使用：`前期证照`、`施工图纸`、`验收时间轴`
- 不再把 `项目列表`、全局 `Dashboard`、独立 `报表`、独立 `监控` 作为主导航命名

### 快捷入口密度规则

- 标题区最多 `1` 个主按钮 + `2` 个次按钮
- 一级总览卡默认只允许整卡点击或 `1` 个局部动作
- 单个信息区最多展示 `3` 个快捷入口
- 列表项右侧最多展示 `2` 个直接动作
- `Dashboard` 和 `CompanyCockpit` 的快捷入口优先承担“进入下一级页面”，不承担新增业务动作
- 不能为了“简洁”删除原有高频入口，只能分层或下沉

## 全局层

### 顶层导航契约（Sidebar）

#### 任务目标
- 将顶层导航收口为公司级双入口：`公司驾驶舱`、`提醒中心`。
- 将项目级入口收口为六大模块，只在进入项目后显示。

#### 本次只允许动的内容
- 侧边栏入口命名、顺序、分组、选中态、折叠态。
- 公司级入口与项目级入口的显示条件。
- 导航层级与入口文案。

#### 本次严禁改动的内容
- 页面背后的数据来源。
- 任何页面的业务逻辑、接口调用、统计口径。
- 进入项目的真实跳转链路和项目 ID 解析逻辑。

#### 需要保留的现有能力
- 从公司级进入项目。
- 从项目内切换各业务模块。
- 历史路由仍可访问。

#### 需要恢复的成熟版能力
- 公司级入口与项目级入口的两层结构。
- 项目内六大模块的稳定命名和排序。

#### 需要合并/下沉/隐藏的入口
- 合并：`项目列表` 表达并入 `公司驾驶舱`
- 隐藏但保留：`Monitoring`、`Settings`、`Reports`、`TeamMembers`
- 下沉：项目内子页不再直接挂在主导航平铺显示

#### 页面结构调整任务
- 将侧边栏拆分为公司级区域和项目级区域。
- 项目未选中时仅展示公司级入口。
- 项目已选中时展示项目级六大模块。

#### 视觉层统一任务
- 统一导航分组标题、选中态、图标尺寸、悬停反馈、留白节奏。
- 统一一级入口和二级入口的字体层级与间距。

#### 路由与导航处理任务
- 导航展示与现有路由解耦，隐藏不等于删除路由。
- 历史入口统一通过兼容跳转或隐藏处理承接。

#### 自测检查项
- 顶层是否只显示 `公司驾驶舱`、`提醒中心`
- 进入项目后是否出现六大模块
- 历史工具页是否从主导航消失但仍可访问

#### 风险提示
- 最容易误伤的是把“隐藏入口”做成“删除路由”。
- 最容易越界的是顺手改 `App.tsx` 的业务跳转逻辑。
