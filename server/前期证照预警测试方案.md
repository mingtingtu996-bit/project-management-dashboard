# 前期证照预警功能测试报告

## 测试目标
验证前期证照过期预警功能是否正常工作，包括：
1. 扫描即将过期的证照（7天内）
2. 扫描已过期的证照
3. 生成正确的预警记录
4. 预警级别正确设置（info/warning/critical）

## 测试方法
由于 Node.js 与 Supabase SDK 在当前环境存在兼容性问题，我们采用以下测试方案：

### 方案 A：手动测试（推荐）
在 Supabase Dashboard 的 SQL Editor 中执行以下步骤：

#### Step 1: 创建测试项目（如果不存在）
```sql
INSERT INTO projects (name, description, status)
VALUES ('测试项目', '用于前期证照预警功能测试', 'active')
RETURNING id;
```

#### Step 2: 创建测试前期证照
```sql
-- 使用上一步获取的 project_id
INSERT INTO pre_milestones (project_id, document_type, document_name, issue_date, expiry_date, status, notes)
VALUES 
  ('PROJECT_ID_HERE', 'TEST_WARNING_1', '即将过期的测试证照（7天后）', NOW(), NOW() + INTERVAL '7 days', 'valid', '即将过期'),
  ('PROJECT_ID_HERE', 'TEST_WARNING_2', '已过期的测试证照（1天前）', NOW(), NOW() - INTERVAL '1 day', 'valid', '已过期'),
  ('PROJECT_ID_HERE', 'TEST_WARNING_3', '正常的测试证照（60天后）', NOW(), NOW() + INTERVAL '60 days', 'valid', '正常');
```

#### Step 3: 检查 warnings 表
```sql
-- 验证是否生成了预警记录
SELECT * FROM warnings 
WHERE title LIKE '%测试证照%'
ORDER BY created_at DESC;
```

#### Step 4: 手动触发定时任务
在服务器代码中，可以通过调用 `preMilestoneWarning()` 函数来手动触发预警服务。

### 方案 B：通过后端 API 测试
如果后端服务器正在运行，可以使用 HTTP API 来测试：

#### 1. 获取项目列表
```bash
curl http://localhost:3001/api/projects
```

#### 2. 创建测试前期证照
```bash
curl -X POST http://localhost:3001/api/pre-milestones \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": "PROJECT_ID_HERE",
    "document_type": "TEST_WARNING_1",
    "document_name": "即将过期的测试证照（7天后）",
    "issue_date": "2026-03-26T00:00:00Z",
    "expiry_date": "2026-04-02T00:00:00Z",
    "status": "valid",
    "notes": "测试预警"
  }'
```

#### 3. 手动触发预警服务
在 server/src/index.ts 或 server/src/routes/jobs.ts 中添加一个临时路由：
```typescript
app.post('/api/test/preMilestone-warning', async (req, res) => {
  try {
    await preMilestoneWarning();
    res.json({ success: true, message: '预警服务执行完成' });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});
```

然后调用：
```bash
curl -X POST http://localhost:3001/api/test/preMilestone-warning
```

#### 4. 查询预警记录
```bash
curl http://localhost:3001/api/warnings?title=测试证照
```

## 预期结果

### 即将过期的证照（7天后）
- 应该生成一条预警记录
- `warning_level` = 'warning'
- `warning_type` = 'expiring_soon'
- `is_acknowledged` = false
- `resolved` = false

### 已过期的证照（1天前）
- 应该生成一条预警记录
- `warning_level` = 'critical'
- `warning_type` = 'expired'
- `is_acknowledged` = false
- `resolved` = false

### 正常的证照（60天后）
- **不应该**生成预警记录

## 已知问题

1. **Node.js 与 Supabase SDK 兼容性问题**
   - 症状：运行测试脚本时出现 "Assertion failed: !(handle->flags & UV_HANDLE_CLOSING)" 错误
   - 原因：可能是 Windows 上的 Node.js 版本与 Supabase SDK 的兼容性问题
   - 解决方案：使用 Supabase Dashboard 或后端 API 进行测试

2. **定时任务调度**
   - 预警服务已在定时任务调度器中注册（每天凌晨 03:00 执行）
   - 确认调度器配置：server/src/index.ts 或 server/src/scheduler.ts

## 附录：完整的 SQL 测试脚本

```sql
-- ============================================
-- 前期证照预警功能完整测试脚本
-- ============================================

-- Step 1: 创建测试项目（如果不存在）
DO $$
DECLARE
  test_project_id UUID;
BEGIN
  -- 查找或创建测试项目
  SELECT id INTO test_project_id 
  FROM projects 
  WHERE name = '测试项目';
  
  IF NOT FOUND THEN
    INSERT INTO projects (name, description, status)
    VALUES ('测试项目', '用于前期证照预警功能测试', 'active')
    RETURNING id INTO test_project_id;
  END IF;
  
  RAISE NOTICE '测试项目ID: %', test_project_id;
  
  -- Step 2: 清理旧测试数据
  DELETE FROM warnings WHERE title LIKE '%测试证照%';
  DELETE FROM pre_milestones WHERE document_type LIKE 'TEST_%';
  
  RAISE NOTICE '已清理旧测试数据';
  
  -- Step 3: 创建测试前期证照
  INSERT INTO pre_milestones (project_id, document_type, document_name, issue_date, expiry_date, status, notes)
  VALUES 
    (test_project_id, 'TEST_WARNING_1', '即将过期的测试证照（7天后）', NOW(), NOW() + INTERVAL '7 days', 'valid', '即将过期'),
    (test_project_id, 'TEST_WARNING_2', '已过期的测试证照（1天前）', NOW(), NOW() - INTERVAL '1 day', 'valid', '已过期'),
    (test_project_id, 'TEST_WARNING_3', '正常的测试证照（60天后）', NOW(), NOW() + INTERVAL '60 days', 'valid', '正常');
  
  RAISE NOTICE '已创建 3 个测试证照';
  
END $$;

-- Step 4: 查询测试证照
SELECT 
  document_name,
  expiry_date,
  expiry_date - NOW() AS days_until_expiry,
  CASE 
    WHEN expiry_date < NOW() THEN '已过期'
    WHEN expiry_date <= NOW() + INTERVAL '7 days' THEN '即将过期'
    ELSE '正常'
  END AS status
FROM pre_milestones
WHERE document_type LIKE 'TEST_%'
ORDER BY expiry_date;

-- Step 5: 手动生成预警（模拟预警服务逻辑）
-- 注意：这一步需要通过后端服务或定时任务来完成
-- 或者使用以下 SQL 手动插入预警记录（仅用于测试）

DO $$
DECLARE
  test_project_id UUID;
  doc RECORD;
  warning_level VARCHAR(20);
  warning_type VARCHAR(50);
  days_until_expiry INTEGER;
BEGIN
  -- 获取测试项目ID
  SELECT id INTO test_project_id 
  FROM projects 
  WHERE name = '测试项目';
  
  -- 遍历即将过期或已过期的证照
  FOR doc IN 
    SELECT * FROM pre_milestones
    WHERE project_id = test_project_id
      AND document_type LIKE 'TEST_%'
      AND expiry_date <= NOW() + INTERVAL '7 days'
      AND status = 'valid'
  LOOP
    days_until_expiry := EXTRACT(DAY FROM (doc.expiry_date - NOW()));
    
    -- 确定预警级别和类型
    IF doc.expiry_date < NOW() THEN
      warning_level := 'critical';
      warning_type := 'expired';
    ELSIF doc.expiry_date <= NOW() + INTERVAL '7 days' THEN
      warning_level := 'warning';
      warning_type := 'expiring_soon';
    ELSE
      warning_level := 'info';
      warning_type := 'expiring_soon';
    END IF;
    
    -- 插入预警记录
    INSERT INTO warnings (
      project_id,
      warning_type,
      warning_level,
      title,
      description,
      is_acknowledged,
      resolved
    ) VALUES (
      doc.project_id,
      warning_type,
      warning_level,
      doc.document_name,
      format('证照类型: %s\n到期日期: %s\n剩余天数: %d天',
        doc.document_type,
        to_char(doc.expiry_date, 'YYYY-MM-DD'),
        days_until_expiry
      ),
      false,
      false
    );
    
    RAISE NOTICE '已生成预警: % (级别: %)', doc.document_name, warning_level;
  END LOOP;
  
END $$;

-- Step 6: 验证预警结果
SELECT 
  title,
  warning_type,
  warning_level,
  is_acknowledged,
  resolved,
  created_at
FROM warnings
WHERE title LIKE '%测试证照%'
ORDER BY created_at DESC;

-- Step 7: 统计预警
SELECT 
  warning_level,
  COUNT(*) as count
FROM warnings
WHERE title LIKE '%测试证照%'
GROUP BY warning_level;
```
