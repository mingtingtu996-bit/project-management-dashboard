## 预警服务配置

### 预警规则
```typescript
const WARNING_CONFIG = {
  // 提前7天预警
  ADVANCE_WARNING_DAYS: 7,
  // 提前30天预警（长期预警）
  LONG_TERM_WARNING_DAYS: 30,
  // 过期后仍持续提醒的天数
  OVERDUE_REMINDER_DAYS: 7
}
```

### 预警级别
- **critical**: 已过期 或 3天内即将过期
- **warning**: 4-7天内即将过期
- **info**: 其他情况（暂未使用）

### 状态过滤
只检查以下状态的证照：
- ✅ 待申请
- ✅ 办理中
- ✅ 已取得

**排除**:
- ❌ 已过期
- ❌ 需延期

## 预警流程

### 1. 扫描即将过期证照
```sql
SELECT * FROM pre_milestones
WHERE status NOT IN ('已完成', '已取消', '已延期')
  AND expiry_date IS NOT NULL
  AND expiry_date <= CURRENT_DATE + INTERVAL '7 days'
ORDER BY expiry_date ASC
```

### 2. 创建/更新预警记录
- 如果已存在相同证照的预警 → 更新现有预警
- 如果不存在 → 创建新预警

### 3. 标记过期证照
```sql
UPDATE pre_milestones
SET status = '已延期'
WHERE expiry_date < CURRENT_DATE
  AND status NOT IN ('已完成', '已取消', '已延期')
```
