# 登录系统实施指南

## 概述

本指南说明如何在房地产工程管理系统中集成登录系统（选项1：修改现有users表）。

## 实施步骤

### 阶段1：数据库准备（已完成）

#### 1. 执行Migration 050

**文件位置**: `server/migrations/050_add_login_fields.sql`

**执行方式**:

1. **通过Supabase Dashboard**:
   - 登录 Supabase Dashboard
   - 进入 SQL Editor
   - 粘贴 `050_add_login_fields.sql` 的全部内容
   - 点击 Run 执行

2. **通过命令行**（如果有psql客户端）:
   ```bash
   psql -h [host] -U [user] -d [database] -f server/migrations/050_add_login_fields.sql
   ```

**Migration内容**:
- 添加 `username`, `password_hash`, `email`, `role` 字段到users表
- 为现有device_id用户生成临时用户名和密码
- 创建默认管理员账户：`admin` / `admin123`
- 启用users表的RLS策略
- 添加索引优化查询性能

**默认管理员账户**:
- 用户名: `admin`
- 密码: `admin123`
- 角色: `owner`

### 阶段2：后端依赖安装

#### 2. 安装npm依赖

**文件位置**: `server/package.json`

**新增加的依赖**:
- `bcrypt`: 密码哈希和验证
- `jsonwebtoken`: JWT令牌生成和验证
- `@types/bcrypt`: bcrypt类型定义
- `@types/jsonwebtoken`: jsonwebtoken类型定义

**安装命令**:
```bash
cd server
npm install bcrypt jsonwebtoken
npm install --save-dev @types/bcrypt @types/jsonwebtoken
```

**注意事项**:
- 如果遇到npm错误，尝试使用 `--legacy-peer-deps` 选项
- 确保Node.js版本 >= 18

### 阶段3：后端代码

#### 3. 认证模块

**创建的文件**:
- `server/src/auth/config.ts` - JWT配置
- `server/src/auth/types.ts` - TypeScript类型定义
- `server/src/auth/jwt.ts` - JWT工具函数
- `server/src/auth/middleware.ts` - 认证中间件
- `server/src/auth/password.ts` - 密码哈希和验证

#### 4. API路由

**创建的文件**:
- `server/src/routes/auth.ts` - 登录API
- `server/src/routes/auth-register.ts` - 注册API
- `server/src/routes/auth-logout.ts` - 登出API
- `server/src/routes/auth-me.ts` - 获取当前用户信息
- `server/src/routes/members.ts` - 成员管理API

#### 5. 环境变量配置

在 `server/.env` 中添加：
```env
# JWT密钥（生产环境必须修改）
JWT_SECRET=your-secret-key-change-in-production

# JWT令牌有效期
JWT_ACCESS_EXPIRES_IN=7d
```

### 阶段4：前端代码

#### 6. 认证Hook

**创建的文件**: `client/src/hooks/useAuth.ts`

**功能**:
- 管理认证状态（isAuthenticated, user, loading）
- 提供login、register、logout方法
- 自动检查登录状态

#### 7. UI组件

**创建的文件**:
- `client/src/components/LoginDialog.tsx` - 登录/注册弹窗
- `client/src/components/UserMenu.tsx` - 用户菜单（备用）

#### 8. 主应用集成

**修改的文件**:
- `client/src/App.tsx` - 集成useAuth Hook
- `client/src/components/layout/Header.tsx` - 添加登录按钮和用户菜单

### 阶段5：配置CORS

#### 9. 配置后端CORS

在 `server/src/index.ts` 中确保CORS配置正确：
```typescript
app.use(cors({
  origin: [
    'http://localhost:5173', // Vite dev server
    'http://localhost:3000', // Production build
  ],
  credentials: true, // 允许携带Cookie
}));
```

#### 10. 配置前端API基础URL

在 `client/.env` 中添加：
```env
VITE_API_URL=http://localhost:3001
```

## 验证步骤

### 1. 数据库验证

检查Migration是否执行成功：
```sql

SELECT column_name, data_type FROM information_schema.columns WHERE table_name = 'users';

-- 查看默认管理员
SELECT * FROM users WHERE username = 'admin';

-- 查看RLS策略
SELECT * FROM pg_policies WHERE tablename = 'users';
```

### 2. 后端验证

启动后端服务器：
```bash
cd server
npm run dev
```

测试API：
```bash

curl -X POST http://localhost:3001/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# 测试获取用户信息（需要先登录获取token）
curl http://localhost:3001/api/auth/me \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 3. 前端验证

启动前端开发服务器：
```bash
cd client
npm run dev
```

测试步骤：
1. 打开浏览器访问 `http://localhost:5173`
2. 点击右上角的登录按钮
3. 使用默认管理员账户登录：`admin` / `admin123`
4. 验证用户信息显示正确
5. 点击退出登录
6. 测试注册新用户功能

## 安全建议

### 1. 修改默认密码

**必须修改默认管理员密码**：
1. 使用admin账户登录
2. 通过数据库直接修改（临时方案）
3. 或实现修改密码功能后通过UI修改

**通过数据库修改**：
```sql
-- 生成新密码的bcrypt哈希（需要使用bcrypt工具或后端API）
-- 然后更新密码
UPDATE users 
SET password_hash = '$2b$10$NEW_HASH_HERE' 
WHERE username = 'admin';
```

### 2. 配置生产环境

**JWT密钥**: 必须使用强随机密钥
```env
JWT_SECRET=$(openssl rand -base64 32)
```

**HTTPS**: 生产环境必须使用HTTPS
**Cookie安全**: 确保 `httpOnly` 和 `secure` 标志已启用

### 3. 权限控制

**项目成员管理**:
- 只有project的owner可以添加/移除成员
- 成员可以查看自己参与的项目
- 使用RLS策略确保数据隔离

## 故障排查

### 问题1：npm install失败

**解决方案**:
```bash
# 清除npm缓存
npm cache clean --force

# 使用--legacy-peer-deps
npm install --legacy-peer-deps

# 或使用yarn/pnpm
yarn install
# 或
pnpm install
```

### 问题2：登录后token无效

**检查项**:
1. JWT_SECRET是否前后端一致
2. Cookie是否正确设置（httpOnly, secure, sameSite）
3. 令牌是否过期（默认7天）

### 问题3：CORS错误

**解决方案**:
1. 检查后端CORS配置是否包含前端URL
2. 确保credentials: true已设置
3. 检查前端是否使用credentials: 'include'发送请求

### 问题4：RLS策略阻止访问

**检查项**:
1. Migration 050是否成功执行
2. users表是否启用了RLS
3. 当前用户是否是项目成员
